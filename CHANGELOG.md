# 更新日志

斗龟场·实时版。格式参照拳头的 patch notes：**按玩家能感知的改动记账**，不是代码流水。

**版本号规则**（`大版本.功能版本.改动序号`）：

| 位 | 什么时候 +1 |
|---|---|
| 第 1 位 | 大阶段（`0` = 测试期，`1` = 正式发布） |
| 第 2 位 | 加了新玩法 / 新系统（例：训龟大师上线） |
| 第 3 位 | **每次有玩家能感知的改动**（用户 2026-07-22：「每一次改动为一个小版本」） |

> ⚠ 改版本号要**同时**改四处，否则门禁 `verify_version` 会红：
> `project.godot` 的 `config/version`、本文件最新条目、`export_presets.cfg`（iOS `short_version` + Android `version/name`）。
> 这条门禁存在的唯一理由：**测试者报 bug 时必须能说清是哪个版本**，四处不一致就说不清了。

## 0.11.1 — 2026-07-22

**修复 · 开局摆位拖不动龟**

- **放置阶段拖自己的龟，龟停在半路、镜头却跑了**（测试反馈：「开局拖动是什么情况」）。原因是上一版为了修「拖视角误触面板」，让相机在判定为拖动时抢走事件——**没考虑到摆位本来就要靠拖动**。现在只要手里抓着龟，镜头就完全让路；点空地拖仍然能移动视角。

## 0.11.0 — 2026-07-22

**新增 · 技能说明分「简明 / 详细」两级**

参照英雄联盟的做法：**简明只给算好的实时数值**（如「造成 210 物理伤害」），**详细才展开公式与比率**（如「(50%护甲 + 50%魔抗) × (1 + 4%×岩层) = 210」）。

- 战斗信息面板的技能段上方加了一个 **「简明 ▸ / 详细 ▾」** 开关，一下子把整个面板（被动 + 所有技能）全切。
- **你的选择会记住**，换场景、换对局都还是这个模式。
- 实测小龟：技能段 101 → 235 字符，被动段 211 → 711 字符。

**修复**

- **同一个面板里两种口径**：此前被动段写死显示详细、技能段写死显示缩略，自相矛盾。现在都听同一个开关。

## 0.10.3 — 2026-07-22

## 0.10.2 — 2026-07-22

**新增 · 训龟大师可以操控了**

- **手机：左下角虚拟摇杆**推着他走。摇杆**半推就是半速**，不是一推到底才动。
- **电脑：WASD 或方向键**。（鼠标那三种操作——拖动平移、点击选中、滚轮缩放——已经排满了，键盘是唯一不打架的通道。）
- **移速 130**。走到战场边缘会停住，不会走出地图。
- 只操控**你自己那个**；对面那个是人机，你的输入碰不到它。
- 他仍然不会被卷进战线：不被主动索敌，也不会被龟挤来挤去。

## 0.10.1 — 2026-07-22

**修复 · 暂停时鼠标失灵**

- **点暂停后光标不动了**（测试反馈：「这是大问题」）。游戏用的是自绘光标（那只绿龟爪），它靠每帧刷新跟随鼠标；暂停会把这个刷新一起停掉，而系统光标又是隐藏的，于是看起来鼠标彻底失灵。现在暂停中光标照常跟手。

> 这和上一版修的「暂停后拖不动镜头」是同一类原因，但**是两个独立的地方** —— 修好镜头那次并没有顺带修好光标。

## 0.10.0 — 2026-07-22

**新增 · 训龟大师（机制部分）**

双方场上各多了一个**训龟大师**，站在自己基地后方角落 —— 他不是战力，是个**场外监视者**：

- **500 生命 / 1 攻击 / 0 双抗**，站着不动。
- **不会被主动索敌**：敌人贴到他脸上也不会转头打他。
- **但吃 AOE**：范围伤害照样落到他身上。
- **受到的所有类型伤害都降为 1**，**真实伤害也不例外**。
- **射程 2000，普攻扔石头造成 1 点物理伤害**（够得到全场）。
- **不计入团灭判定** —— 打不死他也不会让战斗永远打不完。
- **上半场 / 下半场 / 终极战场各自重新登场，血量每场重置**。

> ⚠ **形象还没做**：你要的「像素风的冒险家」尚未定稿，现在暂时用小龟的立绘顶替，游戏启动时会有一条警告提醒。**5 选 1 的手动法术**也还没做（按你说的「法术技能待制作」留到下一步）。

## 0.9.10 — 2026-07-22

**新增 · 新手引导真的能用了**

主菜单的「❓ 教程」此前只是**进一场普通战斗，没有任何引导** —— 引导组件其实早就写好了，但游戏里没有任何地方调用它。现在接上了：

- **战斗界面 4 步**：战斗是全自动的、点龟看详情（**这一步会等你真的点开面板才继续**）、缩放拖动怎么用、打完去哪变强。
- 正常对局不受影响，只有从「❓ 教程」进入时才会显示。

## 0.9.7 — 2026-07-22

**修复 · 被动说明文案**

上一版只订正了「技能」说明，**被动说明漏掉了**（它的详细字段叫 `desc` 不叫 `detail`，检查工具没看这一块）。以代码实测值为准补齐：

- **忍者龟·忍术**：冲击触发范围 500 → **290 码**，冲刺距离 450 → **300 码**。
- **钻石龟·钻石结构**：不是「全队护甲魔抗加成效果额外放大 +50%」，而是**登场时全队（含自己）护甲、魔抗各 +50%** —— 这两句描述的是完全不同的机制。

**内部**

- 差分门禁补上 `passive` 区块，覆盖 112 → 140 对；并新增覆盖自检，将来再漏掉整个字段会直接报错而不是安静通过。
- 顺带订正 4 处过期的代码注释（忍者冲刺的函数头注释一直写着已被废弃的 450 码）。

## 0.9.5 — 2026-07-22

**调整 · 颜色**

- **物理伤害的红色统一了**。此前技能/装备说明里用的是 `#ff6b6b`，而战斗飘字与伤害统计面板用的是 `#ff4444` —— 同一个"物理"两种红。现在全部统一为 `#ff4444`（说明文字里的红字会略深一点，战斗中的观感不变）。

- 新增门禁 `verify_palette`：三处运行时取到的颜色必须一致；`visual_constants` 保留字面量以维持与 Phaser 原型 1:1 的可核对性，但值改漂了会当场红。

## 0.9.4 — 2026-07-22

**修复 · 技能说明文案**

选龟界面（缩略说明）与战斗面板（详细说明）长期各说各的，**以代码实测值为准**逐条订正：

- **小龟·过肩摔**：波及范围 250 → **350 码**；摔落点 60 → **55 码**（两份文案原本一致地都写错了）。
- **石头龟·磐石之躯**：冲击波不是「1%×岩层概率眩晕 1.5 秒」，而是**必定眩晕 2 秒**。
- **石头龟·嘲讽**：收尾砸地的击飞 2 秒 → **1.2 秒**。
- **天使龟·祝福**：攻速加成 +30% → **+50%**；删掉早已不存在的「+30% 龟能充能」；补上漏写的「首次死亡以 25% 最大生命复活一次」。
- **猎人龟·狩猎弹幕**：每箭 30% → **36% 攻击力**（原文案同一句里写着 30% 却算 0.36，自相矛盾）；箭是**随机锁敌并追踪**，不是「直线飞行、可被挡、打最残血」。
- **海盗龟·火炮齐射**：详细说明的「六段 0.17A + 1.7% 生命」是回合制旧口径，实为**六发，每发 0.5A + 2% 目标最大生命**。
- **星际龟·虫洞**：吸引 90 码 → **吸引 150 码 / 捕获 100 码**；伤害不在经过时结算，而是**虫洞抵达地图边界爆炸时**造成。

**内部**

- 新增门禁 `brief_detail_audit`：缩略与详细说明的数值再分家会直接卡住提交。

## 0.9.3 — 2026-07-22

**修复 · 战斗界面操作**（测试人员反馈）

- **暂停时也能拖动、缩放看战场了**。此前点了暂停键鼠标完全没反应。
- **拖动视角不再误开 / 误关单位详情面板**。此前从单位附近起手拖视角会瞬间弹出面板，从空白处起手拖会把已开的面板关掉——单位命中半径 64px，战场上几乎找不到"安全起手点"。
- **手机端拖动速度修正**：此前触屏拖动是设计速度的 2 倍（系统的模拟鼠标事件被重复计算了一次）。
- **双指捏合缩放后抬起一根手指，视角不再突然跳走**。
- 修复鼠标松开落在面板上时，之后不按键移动鼠标镜头也会一直跟着跑。
- 暂停 / 恢复时清掉拖动残留状态，恢复后镜头不再自己漂。

**内部**

- 游戏内、iOS 包、Android 包的版本号统一（此前分别是"无"/`0.9.0`/`1.0`），主菜单右下角显示当前版本。
