# BUG: 多源伤害血条不随飘字逐段下降 (用户实战报, 2026-06-24)

## 症状
多段技能 / AOE 打到人, 伤害飘字一个个分段弹出, 但血条**直接掉到最终值**(全部伤害扣完后)、不随飘字一段段降。用户原话: "血量降到不动, 数字在跳, 感觉血条问题"。"数字在跳" = 命中飘出的伤害数字分段弹; "血量降到不动" = 血条直接到N段后的终值, 没随数字变。

## 复现
- **有些龟都有; AOE 也有**(用户明确: 不是个例)。
- 例: **寒冰龟带装备, AOE 打 3 人, 其中【1 人】出现此情况**(其余 2 人正常)。
  → 那 1 人 = 收到 **AOE 基础伤害 + 装备追伤(proc)** 两个源, 但两源没全被合并。

## 根因 (已定位)
- HP 在 execute 阶段**一次性全扣完**; 逐段下降是 display-only "看起来在掉" (BattleScene.gd:8851)。
- 逐段靠 `_coalesce_multihit_segments` (BattleScene.gd:8855): 只合并【同目标 ≥2 条 `kind=="damage"` 且无现成 segments 的 effect】→ 建 segments(每段 `hp_after` 步血, `_render_one_segment`:8920 飘字+步血同步)。
- **漏网**: 一个目标的多个伤害源若**不是**"≥2 条 kind=='damage' 同目标 effect"(如 AOE 的 1 条 damage + 装备 proc 走别的 kind / handler 自己单独 `_spawn_float_text`+`_refresh_slot(目标)` 刷到终值), 就**不会合并** → 血条第一下就刷到终值, 飘字仍分段 = 症状。

## 修法方向 (通用, 非逐技能补)
- 让血条对【该目标本次所有伤害飘字】逐段跟随, **不限于** ≥2 条同 kind damage 的合并。
- 即: 把"血条步降"绑定到**飘字 spawn** — 任何来源(AOE 基础/装备 proc/审判/DoT/多段)的伤害飘字都把"显示血量"步降, 收敛到真实终值。防回弹守卫 (hp_bar.gd:60) 已在, 不会被抬高。
- **视觉回归高发区**(历史教训: 逐段飘字曾引血条回弹/乌龟卡红) → **必须 F5 眼验**, 单测只能保逻辑不回归、保不了画面。

## 实装计划
- 派 agent **"大找"**: 审所有多源伤害展示路径(AOE 基础 + 装备 proc / 天使审判 / 多段技能 / DoT)怎么发飘字 + 刷血 → 做**通用修**(血条跟飘字步降) → 跑测(基线不降) + 列出需 F5 的点。
