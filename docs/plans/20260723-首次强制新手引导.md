## 4. 方案

### 4-a 首次检测（低风险）
- `GameState` 加 `var onboarded: bool = false`，进 `save()`/`load()` 的字段表。
- `MainMenuScene._ready()`：`if not GameState.onboarded and not 有存档: _start_onboarding()`。
- 走完教学 → `GameState.onboarded = true; GameState.save()`，之后永不再触发。
- **开发/测试开关**：`env ONBOARD=1` 强制触发、`ONBOARD=0` 强制跳过（否则本机跑一次就再也测不到，等于没写——EQDEMO 那次的教训）。

### 4-b 强制教学对局（复用现成）
- 复用 `_on_tutorial` 那套进战斗的逻辑，但**跳过"是否开始"确认框**（首次直接进）。
- 预设一套固定阵容（3 只上手简单的龟，写死 id），不让新手在空白里选。
- 第一次**不给"跳过教程"按钮**（`TutorialGuide` 加 `mandatory` 参数）。

### 4-c 高亮指引（本方案的核心新增）
`TutorialGuide` 现在只有黄框提示条。加一层**遮罩 + 挖洞高亮**：
- 全屏半透明暗幕（`ColorRect`，挡住非目标区域的点击）。
- 当前步骤指定一个**目标控件/屏幕矩形**，暗幕在那里"挖个洞"（亮框 + 可点），其余压暗。
- 步骤数据 `tutorial-steps.json` 每步加 `highlight` 字段：目标控件的锚点（如 `"team_roster"` / `"start_button"`）。
- 场景侧提供 `_tutorial_anchor(name) -> Rect2`，把名字换成屏幕矩形（选龟界面的头像排/开始按钮等）。

### 4-d 文案你来定
`tutorial-steps.json` 的 8 步文案是我写的。做完给你过一遍，你改哪句我改数据文件（不碰代码）。

## 6. 验收标准（怎么证明它会 FAIL）

门禁 `verify_onboarding`（行为级）：
1. `GameState.onboarded=false` 且无存档 → 主菜单 `_ready` 真的走了 `_start_onboarding`（不是只设了 flag）
2. 走完教学 → `onboarded` 变 true 且 `save()` 被调
3. `onboarded=true` → 主菜单**不再**触发引导（正常进菜单）
4. 首次教学的 `TutorialGuide` **没有**"跳过"按钮（`mandatory=true`）
5. 每个 `highlight` 锚点都能解析出一个非零 Rect2（否则暗幕挖洞挖了个空 → 高亮了寂寞）
6. 反向验证：把首次检测条件写死 false → 第 1 条红；去掉 mandatory → 第 4 条红

**★分母**：`tutorial-steps.json` 里带 `highlight` 的步数 > 0（否则等于没做高亮）。
