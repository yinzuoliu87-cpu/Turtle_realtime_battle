# 方案书 · 技能/装备描述 内联属性图标

**日期** 2026-07-24 ·**状态** 已实现 ·**版本** 0.15.5

## 需求
用户：技能/装备描述里，属性词前面要像"人家"那样带一枚属性图标（参考 `codex_xiaogui.png` 的属性面板：图标紧贴属性、同高）。

## 决策（均已与用户拍板）
1. **选 A**：只给【真属性】加图标（攻击力/护甲/魔抗/最大生命值/暴击率/暴击伤害/生命偷取/护盾/闪避/攻击速度/移动速度/射程）。**伤害类型词（物理/魔法/真实伤害）、DoT/控制词（灼烧/中毒/眩晕/诅咒…）不加**——否则一句里图标太密。
2. **术语先补全再统一**：`最大生命`→`最大生命值`（45 处）；长短同义混用**全用完整长写**：`攻速`→`攻击速度`、`移速`→`移动速度`、`吸血`→`生命偷取`、少数 `防御`→`护甲`。`baseStats` 紧凑属性块（`攻/护穿/吸血` 那种）与技能名"防御"**不动**。
3. **图标随字号缩放（选 C）**：图标高 = 字号 × 1.15，各显示场合与文字同高居中。

## 实现
- 图标源：统一那套 `assets/sprites/stats/*-icon.png`（12 属性·32×32，v0.15.4 做的）。
- `scripts/engine/skill_text.gd`：
  - `KEYWORD_RULES` 每条可带第 3 项 = icon-key；关键词自动上色时，有 icon 的在词【前】插 `<img src=".../key-icon.png"/>`。
  - `html_to_bbcode(html, icon_px)`：把 `<img src>` 转 `[img=icon_px]path[/img]`。
  - `render_bbcode(template,f,s, font_px=0)`：`icon_px = font_px×1.15`（0=默认 16）。
- 调用处传字号：CodexScene（技能 brief/detail/被动 13、装备 14）、TeamSelectScene tooltip（14）。
- **全 28 龟 + 全装备自动生效**（都走 SkillText，无需逐条改数据）。
- 门禁 `tests/verify_desc_icons.gd`：断言真属性前有对应图标、伤害类型不加、无 `<img>` 泄漏、`[img]` 语法。

## 已知风险与未决点
- **战斗内技能提示**走 `render_plain`（纯文本，无 bbcode）→ 那里**没有**图标，只有图鉴/选龟的 bbcode 描述有。若要战斗内也有，得把那条改成 RichTextLabel（未做，待用户要）。
- **图标尺寸随字号是"×1.15"经验值**：字号 13→15、18→21，实测贴。若某场合觉得偏大/小，调这个系数或那处传的 font_px。
- **TeamSelect 的 tooltip** 若是原生 `tooltip_text`（非 RichTextLabel）则 `[img]`/`[color]` 都不解析——但这与现有 `[color]` 行为**一致**（我这步没引入新问题；原本能显色就能显图标）。
- **`baseStats` 未加图标**：它是 `+攻15/护穿3/吸血5%` 那种紧凑缩写块，风格与散文描述不同，加图标反而乱。若用户要，另议。
- **数据审计**：术语改动只动文字不动数字，`tooltip_number_audit`/`brief_detail_audit` 按数字三元组对账，不受影响（门禁验证）。
