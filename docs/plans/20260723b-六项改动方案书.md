## 1. 需求原文（逐字）

1. DOT 伤害显示优化：现状如一个单位被诅咒后头顶会快速跳 1，灼烧流血等也是一秒头上浮动一个数字，改为收到 DOT 后头顶出现一个数值不断变大的对应颜色的数字，表示在这个 dot 下逐渐累积受到伤害，数字视觉大小也遵循规则越来越大，该 dot 结束后这个数字遵循现在的跳走规则；同一时间收到两个不同伤害类型的 dot 则左右错开，如果同一时间有灼烧和中毒（同一伤害类型）则算在同一个数字里不断变大，把边界情况考虑周全；诅咒先不要触发音效。
2. 测试人反馈地图装饰比较猎奇，感觉是密集和很多相同装饰引发不适，思考该怎么设计。 → 澄清：**密+素材都有问题，可以多创点美术素材**。
3. 法术技能圆盘放右侧，先设计 1 个技能钩锁：玩家按住右边技能图标后拖动改变方向，PC 端额外设计，射程 600 码，朝方向甩出钩锁（参考 LoL 锤石 Q），钩住第一个敌人，眩晕 4 秒并在这 4 秒内每秒朝训龟大师方向拖拽 70 码，这 4 秒内受到的伤害增加 25%，CD 20 秒。 → 澄清：**新美术不复用；PC 学 LoL 按 Q 朝鼠标方向发射；空放（没钩到人）返还 10 秒剩余 CD；眩晕吃韧性；双方大师都会做场外技能援助（Botworld 式），快照大师也要像真人一样乱动。**
4. 训龟大师仍会被很多技能锁到（护盾均分、赛博浮游炮、饮血斩击等），大师是场上监视者，很多情况不该主动参战，只应被 AOE 溅射/波及（如龟派气波）；赛博龟死后被动机甲也有类似问题（被忍者被动击飞）。彻底解决。 → 澄清：**规则=定向技能选目标时不选大师，但路径/范围扫到大师照样造成伤害（算场外援助）；机甲问题特指 5 秒组装演出期间不该主动攻击、也不该被忍者龟等直接冲击；逐技人工仔细判断。**
5. 图鉴-龟区域：现在没有龟之间的羁绊，不要物理法术了，显示属性加移速和攻击速度，技能区仔细排版。 → 澄清：**守护/元素等 tag 全是凑羁绊的、羁绊已废 → 全删；攻速有等级倍率（我原判断错，已改）；技能描述照 LoL 现行文案风格重写、并做技能框排版（研究 40+ 条现行 LoL 描述）。**
6. 训龟大师现在在摆放阶段不要直接拖动，而且此阶段为什么一直在做投掷动作。

## 4. 方案（改哪些、为什么、穷举）

### 4.1 DOT 累积数字
- 每单位 `u["_dot_float"]={}`（`:3380-3382` 初始化），**key=伤害类型桶(mag/phy/tru)**，value=常驻 Label 句柄 + 累积值 + 本桶最晚 until。
- `_apply_damage` 加"DOT 累积模式"标志：DOT 时**跳过** `:8989` 即时弹字，改累加进对应桶 Label（`text=累积总数`、字号走 `_float_size`）；普攻不受影响。同标志跳过 `:8994 _sfx_hit`（静音诅咒/DOT tick）。
- **左右错开**：同单位多桶并存 → 按桶给 x 偏移（mag 中/phy 左/tru 右）。
- **结束才跳走**：层数 DoT `_tick_dot_stacks:18116`（`ds.erase`）、诅咒 `_tick_effects:4991/4999` 过期点 → 把该桶 Label 转交 `_float_text` 的 `kind=="damage"` 弹射分支（`:9475-9489`+`_dmg_float_step:9504`）飞走。
- 颜色取桶代表色（mag 蓝/phy 红/tru 紫）。真火变体按**实际结算类型**（tru）归桶。字号封顶防糊屏。单位死亡清理常驻数字。
- **穷举改动点**：`_apply_damage:8956`（拦弹字+静音）、`_tick_dot_stacks:18088`（累加+结束钩子）、`_tick_effects` flat 段 `:4989-5000`（诅咒累加+结束钩子）、`_float_text/_dmg_float_step`（复用）、单位字段初始化 `:3380`。

### 4.2 地图装饰
- **补素材**：PixelLab 生成一批新水下装饰（不同形态珊瑚/贝壳/海星/海胆/多种海草等，柔和少刺眼发光），扩 `:992`/`:1779` 调色板。
- **调参**：`:1001` 概率 0.58→~0.3；`:994` step 80→~120；`:993` mg 288→~200；`:992` 去重复加权项。
- **防扎堆**：`:996-1011` 放置循环记录已放点，最小间距（~70px）跳过。
- **破一致感**：生成处 `:1005-1009`/`:1788-1792`（注入点 `_map_billboard:1713`）加随机水平镜像 + 明暗/色相抖动（现 `modulate` 死值）。
- **减层**：系统 B 减量/去；远景 `:1844` `range(22)` 调小。
- **出前后对比图**给用户判是否够。

### 4.3 法术圆盘 + 钩锁
- **右侧圆盘 UI**（挂 `_ui_layer:1284`）：仿 `virtual_joystick.gd` 的 hold-drag `Control`，`mouse_filter=STOP` 隔离，按住图标出**方向指示器+600 码射程扇形**，松手回调方向。
- **PC**：按 **Q** → 朝**鼠标方向**发射（LoL skillshot）。
- **朝方向钩子弹道**：发射用 `_eq_first_in_line`（**加 600 码上限**）预判首个命中；命中→走"钩住+拉回"演出（借鉴海盗钩索结构，**新钩子素材**）；未命中→钩爪飞到 600 码尽头空收。
- **命中效果**：`_stun(hit,4.0,"hook")`（**吃韧性**，`no_tenacity=false`）；被钩单位存 `_hook_pull_until=_t+4`/`_hook_next_pull`，每秒朝大师 `_my_trainer:3510` 拉 70 码（借鉴大熊/海盗 pull）；`hook_vuln_until=_t+4` 接进 `_mitigate_incoming` 放大 `d*=1.25`。
- **CD**：`_hook_cd=20`（每帧扣减，图标冷却蒙版）；**空放返还 10 秒**（发射后没命中 → `_hook_cd = max(0, _hook_cd-10)`，即空放实际 10s）。
- **双方大师**：玩家方走圆盘/Q；对面(快照)大师加**施法 AI**（选方向甩钩）+ **移动 AI**（像真人乱动/走位）。
- **结算逻辑抽出可测**（照 `_pirate_grapple_hit` 抽法，不依赖圆盘/演出 tween）。
- **穷举新增**：圆盘 UI 控件、Q 键输入、朝方向弹道分支、玩家/AI 施法入口、持续拉拽+vuln tick、`_hook_cd` 字段、对面大师移动+施法 AI、新钩子素材。

### 4.4 训龟大师集中闸门
- 新建 `_pick_enemies_of(u)` = `_enemies_of(u)` 排除 `is_trainer` + `_is_untargetable(o)`。**所有"随机/最近/单体指向"站点改走它**：`:19177`(浮游炮)、`:22591`(饮血)、`:18454`(忍者被动)、`:5481`(雷鸣)、`:12109`(猎人处决)。
- **真 AOE 保持 `_enemies_of`**（龟派气波、激光竖劈 `_eq_laser_chop:22414`、扇形等）→ 大师照样被扫到吃 1 点。
- 护盾均分排除 trainer（`_allies_of` 加"含监视者?"参数，或就地 `:5459`/`:17931` 过滤）。
- 机甲组装期：忍者被动改走含 `_is_untargetable` 的新闸门 → 尊重 `untargetable_until` 不被击飞；确认 `_slam:19300` 是否锁住机甲攻击，没锁则补组装门。
- **门禁审计**：遍历所有"从敌人列表单取一个"的用法必须走 `_pick_enemies_of`。
- **穷举**：需逐个技能列"AOE vs 定向"核对表（见风险 R1）。

### 4.5 图鉴龟区域
- 删 tags 块 `:938-946`（全 10 种标签）。
- 属性 `:951` 加：移速（读 `:96` 表 move_spd，**不乘倍率**）、攻速（`1/atk_interval`，**按 `1+0.02*(lv-1)` 等级缩放**，不乘 rarity_mult——与战斗 `:3424` 同源）。★加门禁核对图鉴移速/攻速 = `:96` 表同源。

## 附 · LoL 现行技能描述文案规范（研究 12 英雄 40+ 条，重点近年重做）

**写什么**：伤害(带类型+颜色)、成长加成 `(+120% AP)`、射程、**位移/冲刺/击退距离**、控制类型+时长、减速%+时长、CD/消耗、持续/层数/命中上限、几何差异效果(中心 vs 边缘)。
**不写什么**：**命中判定框宽度/码数**(龟派气波不写多宽)、弹道速度、内部 tick 间隔、前后摇帧、公式推导。范围靠**特效/地面施法指示器**表达。
**风格**：动词开头、主动语态、讲"结果"非"机制"；条件句串效果("命中敌人则…并…")；控制词(眩晕/束缚/减速/恐惧/击飞/击退/勾拽/护盾/治疗)当**加粗关键词**；可变数值标来源("随距离""随等级")。
**格式**：分级斜杠 `60/95/130`；加成括号内联紧跟基础值；伤害类型色(物理橙/魔法蓝青/真实白/治疗盾绿)且**全局一致、与图鉴属性面板同色**；默认只显当前等级值，分级/公式 hover 展开。
**技能框版式（对真实截图核对，非纯文字推断）**：技能框有**简略/详细两版**（默认简略，按 Shift 或设置常开切详细）：
```
[图标] [键]技能名                 CD
                                消耗
主描述：造成 40(+4🗡)物理伤害…    ← 数值按类型着色(物理橙/魔法蓝青/真实白/治疗盾绿)
────────────────────────────
★灰色斜体行：特殊交互/边界说明     ← (如"普攻减1秒CD""可指向任意目标并触发on-hit")
  简略版1行、详细版可多行讲机制/边界
分级数值网格 Damage [40/70/100…]  ← ★仅详细版:每档数值斜杠分级
Press [Shift] to show more info    ← 简略↔详细切换提示
```
**★关键补漏（我初版文字研究漏了，现已核对真图 + 现行 wiki 全文）**：主描述**之下、网格之上**有一段**灰色斜体的"特殊/边界说明"**（现行 wiki 里标 `Special:`）——讲特殊交互、边界情况、隐藏机制。现行(2023-2024 新英雄)实例：
- Senna Q(图)：`普攻减1秒CD` + `cast range 随攻击距离、cast time 随攻速、可指向任意目标并触发on-hit`
- Ambessa Q：`Special: 对野怪造成 75 额外物理伤害(按等级 100-300 血量比例封顶)`
- Briar Q：`Special: 重置普攻计时；可指向野怪植物/守卫/隐形陷阱，但不能指向建筑`

龟技能框照做：**主描述讲主效果 → 灰字段(Special)专门放"边界/特殊情况"**（如龟派气波"贴脸能不能打出/穿不穿墙/对蛋是否生效"这类）。简略版给够用的，详细版(Shift 展开)再给分级网格 + 完整机制灰字。**这一层是让玩家真看懂边界的关键，不能只写主效果。**

### 附.2 · 28 张真·游戏内技能框观察（`C:\tmp\lol_refs2\`：Senna/K'Sante/Bel'Veth/Nilah/Viego/Gwen 全套 QWER+被动，逐张核实为真客户端截图，部分带 PBE `DEBUG-TEMP` 行为铁证）

对着真图，一个技能框自上而下的**完整层级**（比纯文字总结精确）：
1. **抬头**：图标 + `[键]技能名`（左）｜ 冷却 + 消耗（右，No Cost 也照写）｜ 大招标 `Ultimate`。
2. **`Passive:` / `Active:` 粗体小标题**分段（有被动的技能，如 Bel'Veth R、Nilah Q、Viego Q）。
3. **主描述**：动词开头大白话。数值内联**"总值 = 基础 + 成长"**写法，如 K'Sante Q `80 = (50 +40%🗡 +30% bonus🛡 +30% bonus🔮)`、Bel'Veth R `150 = (150 +100%AP)`——总值在前、括号拆成长，全部按类型着色。
4. **项目符 `•` 列表**（多项并列效果，如 Bel'Veth 真身进化：`•100 max Health •25 Move Speed •50 Attack range …`）。
5. **金/彩色条件行**（如 K'Sante Q `All Out: 冷却 -1 秒且不再减速`）——特殊状态下的变化，用非灰的高亮色。
6. **★灰色斜体 Special 段**（1~多行，网格之上）：边界/隐藏机制。如 K'Sante Q `CD 随双抗降低、cast time 随双血降低`；Bel'Veth R `Void Remora 按 70% 血/110% 攻计入小兵；Slow 消耗虚空珊瑚时爬升到 99%`。
7. **分级数值网格**（仅详细版/Shift）：**每个数值一行**，斜杠分级，如 `Damage [50/75/100/125/150]`、`Attack Speed [10%/15%/20%]`。大招 3 档、普通技 5 档。
8. **底部**：简略版 `Press [Shift] to show more info` / `Click to level up`。

**关键词高亮**（真图里加粗+着色的词）：Slow/Stun/Pull/Root/Fear/Knock/Shield/Heal/True damage 等。龟技能框直接照这 8 层搭；**灰字 Special 段是我初版漏的、也是让玩家看懂边界的核心**。
