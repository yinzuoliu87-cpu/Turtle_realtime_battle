## 1. 训龟大师技能装配系统（架构基础 · 先做这个）

### 1.1 现状
- 训龟大师现在**写死**两件事：扔石头普攻（`_tick_trainer_attacks` / `_fire_trainer_rock`）+ 钩锁主动（`_cast_hook`，Q / 圆盘触发）。
- 没有"装配"概念——技能不是数据、不能选。

### 1.2 目标数据模型（GameState）
新增局外持久字段（存档）：
```
var trainer_appearance: String = "default"     # 形象 id（对应一张立绘）
var trainer_passive: String = ""               # 被动技能 id（可空=无被动）
var trainer_active: String = "hook"            # 已装配的【单个】主动技能 id（已定：1 槽）
```
- **主动槽位数＝1（已定）**：右下只一个法术圆盘。
- 技能 id 池：被动 `{magic_stone}`（可空）；主动 `{hook, fury_potion, whistle, glacier}` 里选 1。
- 战斗侧 `_make_unit(TRAINER_ID…)` 读 GameState 把 loadout 写进大师单位字典（`u["_tr_passive"]`, `u["_tr_actives"]`）。
- **敌方/快照大师**：快照里带对手的 loadout（`_dual_foe_lane` 的 ghost/bot 需带 trainer 配置）；冷启动 bot 给默认 loadout。

### 1.3 法术圆盘泛化
- **单主动槽（已定）**：右下**一个**法术圆盘，持有 `skill_id + icon + cd`；点击/按 **Q**（已定，唯一键）路由到 `_cast_active(u, skill_id, aim)`。
- `_cast_active` 是统一入口，按 skill_id 分派到各技能的施放函数；冷却、空放返还等各技能自带参数。
- 现 `spell_disc.gd` 只认钩锁 → 泛化成读 `skill_id` 的通用圆盘。

### 1.4 实施前需核实
- 快照数据结构 `_dual_foe_lane` / `_dl_snapshot_survivors` 是否方便塞 trainer loadout。
- `_make_unit` 里训龟大师分支当前初始化了哪些字段（避免重复/漏）。

## 2. 五个技能规格（含特效 / 交互）

> 数值以用户口述为准；**代码是终审**，实现后所有数值进 verify 门禁。演出一律**结算与 tween 解耦**（CLAUDE.md §3.5）。

### 2.1 钩锁 hook（已落地 0.14.0，纳入池）
- 射程 600 / 眩晕 4s（吃韧性）/ 每秒拖 70 码 / 受伤 ×1.25 / CD20，空放 CD10。已有门禁 `verify_hook`。

### 2.2 魔法石 magic_stone（被动）
- **效果**：大师普攻**命中时**——
  1. 附带 **2% 目标最大生命值**的**魔法伤害**（叠加在扔石头的伤害上；走 `_apply_damage_from` 魔法路径）。
  2. 大师自身获得 **+5% 攻速**，**可无限叠加**，**持续到本场战斗结束**。
- **关键坑**：攻速叠加"持续到本场结束"＝**按本场战斗计时**，必须存 `t0`（不是用全局 `_t`，CLAUDE.md §3.4），或干脆用"换场清零"的计数字段（`u["_ms_stacks"]`，换路/新战场重置）。
- **演出**：命中时大师身上一粒紫色魔法石微光；攻速层数可在大师头顶叠个小数字（可选）。
- **实施前需核实**：训龟大师普攻当前伤害结算走哪条路径（扔石头 `_fire_trainer_rock` → 命中回调），把 on-hit 钩子挂那里。

### 2.3 怒火药水 fury_potion（主动，CD 16s，点目标 700 码）
- **交互**：
  - **PC**：按键（E?）进入**瞄准态**→ 鼠标显示 700 码**射程圈** + 落点 300 码**预览圈**→ 左键确认投掷（超射程则夹到 700 边缘或取消，**未决点③**）。
  - **手机**：点圆盘→进入瞄准态→**拖动/点击**战场落点（射程圈跟随）→ 抬手投掷。
- **效果**：大师投出**怒火药水**（**用素材原图**，抛物线）→ 落点 **300 码内友军**获得 **5 秒**：`+30% 攻速`、`+25% 龟能充能速率`、`+25% 移动速度`。
- **演出**：药水抛物线飞行（复用 `_fire_trainer_rock` 抛物线逻辑）→ 落地红橙 splash + 地面短暂怒火圈 → 受益友军脚下红色 buff 光环 5s。
- **实施前需核实**：龟能充能速率字段（`energy` 增速在哪调）、移速/攻速 buff 用 `_buff()` 积木是否够。

### 2.4 训龟大师的口哨 whistle（主动，CD 14s，无目标·随机 3 选 1）
- 施放时**随机**选下面 1 个释放（`randf` 三分；无瞄准）：
  1. **临时血量**：随机 1 名我方单位获得 **700 临时最大生命值**，持续 5s。**到期＝按比例削**（已定）：当前血 ×（旧上限 / 新上限），即掉血按比例缩，不直接砍到新上限。
  2. **灵体小龟**：召唤 1 只**蓝色幽灵小龟**（新建模），对**最近的一个敌人**放一道**龟派气波**（复用小龟 `basicChiWave` 素材 + 击飞）：沿途敌人**击飞 + 200 物理伤害 + 削减 30% 护甲持续 5s**。
  3. **狂暴**：随机 1 名友军 **+20% 攻击力 + 20% 吸血，持续 4s，期间免疫死亡**（免死＝血量夹在 ≥1，参考凤凰/免死类实现）。
- **演出**：口哨音符图标从大师头顶冒出 → 按分支各自演出（血量＝绿光柱；灵体小龟＝蓝幽灵入场放气波；狂暴＝红色战意光环）。
- **实施前需核实**：`basicChiWave` 施放函数名与复用方式；"临时最大生命值"与"免疫死亡"项目里是否已有等价积木（诅咒/凤凰/护盾），能否复用。

### 2.5 冰川 glacier（主动，CD 17s，方向 500 码）
- **交互**：
  - **PC**：按键 → 朝**鼠标方向**释放（像钩锁，但生成的是**地面带**而非弹体）。
  - **手机**：点圆盘 → 拖动定朝向 → 抬手释放（交互统一学 Wild Rift，见 §9-3）。
- **效果**：沿方向生成一道**冰川**（长 500 码的地面带），**持续 6s**；站在冰川上的**敌人** `-40% 移速` + `承受 +20% 额外伤害`。
- **实现**：**地面持续区域**——直接复用 `_tick_lava_zones`（熔岩池）那套**周期区域**机制（每帧判定谁站在区域内 → 施加减速 + 易伤标记）。易伤 +20% 走 `_mitigate_incoming`（和钩锁 `hook_vuln_until` 同一处加一个 `glacier_vuln_until`）。
- **演出**：一条蓝白冰川从大师脚下沿方向铺开（**新素材：冰川地面带**）+ 站上去的敌人脚下寒气。
- **实施前需核实**：`_tick_lava_zones` 的区域数据结构（形状是圆还是带？带状可能要新增矩形判定）、减速字段。

## 4. 装备削弱

> **真事实源是 `P2RT.STATS` / P2RT 逻辑**（CLAUDE.md §1），`baseStats1` 只是展示镜像，两处都要同步；文案数值过 `tooltip_number_audit`。

### 4.1 龙蛋 p2eq_024
- **现状**：装备即 3 层吐息（`_p2DragonStacks=3`，`P2RT` line 269）。
- **改**：施加的**灼烧层数改为固定 30/45/70 层**（按星级 1/2/3）。
- **实施前需核实**：龙蛋的"吐息→灼烧"在哪应用层数（是每次吐息叠 N 层，还是装备时给 N 层）；`_p2DragonStacks=3` 与"灼烧层数"是不是同一个量（可能 3＝吐息**次数**，灼烧层数另算——**务必先探针打印真实层数**再改，别照名字猜）。

### 4.2 暴君之牙 p2eq_004
- **现状**：牙伤害 15/30/90（扁平）；斩杀线 = base + 暴击×0.1（`P2RT` line 736）。
- **改**：
  - 牙伤害 → **1 / 1.8 / 4 × ATK**（改为吃攻击力，不再扁平）。
  - 斩杀线 → **4 / 6 / 12% ×（1 + 暴击率）**。
- **改动隔离性（已核实 2026-07-23）**：P2RT line 695-703 有一条"弓箭[神射手]羁绊复用暴君之牙处决"的分支，但它靠 `_archerExecBase`，而该字段**只在 `phase2_schools.gd::apply_team_start`（回合制羁绊系统）里设**，**实时版主场景根本不调 `apply_team_start`、全文无"羁绊/神射手"字样**——所以这条分支在实时版**永不触发**（死代码）。改暴君之牙 line 736 的斩杀线**只影响暴君之牙自己**，不牵连任何羁绊。
  > 教训（CLAUDE.md §5）：`scripts/engine/` 是**回合制旧引擎**，它的羁绊/类型代码在实时版是平行死代码，别当活的。

## 5. 大轮机制（8 命淘汰 → 局外锁定态）

### 5.1 现状
- `GameState.hearts = 8`（注释："输-1，0=淘汰；玩法在阶段4"）——**淘汰后的锁定玩法从没做**，正是本需求。
- 商店已有**锁定范式**：`season_total_battles<=0 → shop_locked`（灰显 + 🔒角标 + 仍可点→toast）。**直接复用这套**。

### 5.2 规则
- `hearts == 0` → 进入**淘汰锁定态**：
  - **锁**：匹配（开始战斗）、商店。
  - **背包**：可进，但**锁掉**：装备/卸下、换位调整、前后排切换；**点击装备仍可查看效果**（只读浏览）。
  - **解锁**：只有**重置存档**才开启新的大轮。
- **实施前需核实**：
  - 输一场 `hearts-=1` 的扣命在哪结算（结算流程）；确认到 0 会停在这。
  - 背包场景（`InventoryScene.gd`）里装备/卸下/换位/前后排各是哪个交互入口——要逐个加 `if eliminated: 只读`。
  - "重置存档"入口在哪（设置里？），确认它清 hearts 回 8 + 清淘汰态。
