## 二、角色AI / Botworld 战斗规格 (用户逐帧分析, 当实现标准)
- 节拍: 发现→移动→进射程→**停**→面向→前摇(windup)→出手→命中反馈→后摇(recover)→重新寻敌→移动。**移动与攻击/施法互斥**。
- 移动: 进射程**立即停**(不边走边打); 不每帧重算路径(不来回晃)。 [✅ 状态机已实现互斥+停]
- 普攻反馈: **不打断敌人动作**(不硬直/不击退普攻); 反馈全靠 **命中特效(Hit Spark星形+Impact Ring圆环, 0.08~0.15s) + 伤害数字 + 打击音**。 [✅ 命中特效已加(`_hit_spark`)+伤害数字(#1)+音效]
- 施法锁: 施法期间不移动/不重寻路/不切目标, 放完才恢复。 [✅ windup/recover 站定]
- 后摇: 放完不立即移动, 短暂 recover。 [✅]
- 状态机: Idle→Search→Move→Face→Windup→Attack/Cast→Recover→Search→Move, 严格切换不并行。 [✅ 已建]
- **后续可整理《Botworld战斗设计还原文档》**(AI状态机/技能/普攻反馈/索敌仇恨/寻路站位/镜头节奏) 当规范。
