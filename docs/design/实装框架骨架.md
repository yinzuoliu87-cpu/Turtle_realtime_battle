## §1 选技系统: 4选1 → 3选1 接线改造
**现状**(代码): `_resolve_chosen_index`(L5934) 从 `skillPool[1..4]` 选 **1** 个, `GameState.loadouts[id]` = 选中索引(默认1); 被动型走 `PASSIVE_SKILL_TYPES`(L5930); 未实装的技回落默认(L5950 gate).
**改造**:
- 每龟 `pets.json` 的 `skillPool` **收敛成正好 4 条**: `[0]=普攻(physical/magic)` + `[1][2][3]=最终3选1三技`. 多余旧条目删除.
- 选技UI(CodexScene/选龟) 显示 idx1/2/3 三选一; `loadouts[id]` 存选中的 1 个(int).
- 被动型3选1候选(如龟壳吸收前的强化觉醒已并被动、竹叶强化生长) → 进 `PASSIVE_SKILL_TYPES`, 在 `_apply_spawn_passives` 按是否选中施加.
- 主动型 → 进 `_resolve_active_skills` 轮转(龟能满放).
- **删除 L5950 "未实装回落默认" gate**(实装全部补齐后不再需要, 留着会掩盖漏实装).

## §2 干净收口 — 旧 type/描述清除(对应用户"不用的描述"痛点)
每龟重实装**完**必做收口, 缺一不可:
1. `pets.json` 该龟 `skillPool` 只留 普攻+最终3技; `detail`/`effectDesc`/`brief` 只留最终设计文案, 旧文案删.
2. `_IMPL_SKILLS`(L5982) 删掉该龟不再用的旧 type; 加最终技 type.
3. `_do_skill` match(L6148) 删旧 `_sk_*` 分支; 接最终技.
4. `_sk_*` 旧函数体: 不再被任何龟引用的 → 删(grep 确认 0 调用).
5. `SkillEnergy`(skill_energy.gd) 删旧 type 龟能, 加最终技龟能.
6. 图鉴/信息面板 走同一 pets.json → 自动同步; 抽查显示无旧技.
**收口验证**: 每龟实装完 `grep -n "<该龟所有旧type>" 全代码+pets.json` = **0 残留** 才算过.
已知旧 type 清除样例: soulReap, headlessSoulStrike, hidingCommand(弃), gamblerBet(错误映射到万能牌), 旧ninjaShuriken(飞镖remap), ghostEnhancedCurse(强化怨灵移除) 等.

## §4 测试 harness (验收标准: headless必测 + F5眼验清单)
**headless 必测(每龟, 自动断言, 过不了不算完):**
- `bash run-tests.sh` 全绿(新增技的单测放 tests/); 新类先 `--import`.
- parse: `--headless --quiet` 载入场景零报错(level1 + level5).
- 技能触发: REVIEW_DEMO + `REVIEW_TURTLE`/`REVIEW_SKILL_IDX` 指定该龟该技, 跑战确认技真放出(打印伤害/buff断言).
- 数值断言: 关键系数(如虚化-90%物理、龟能cost、DoT层)用临时打印或单测核对设计值, 验完删打印.
- 无崩: 跑满一场(含濒死/召唤/变身/隐身路径)无 error.
**每龟 F5 眼验清单(留给用户, 我不代验视觉):** 每龟实装完在 `docs/design/F5验收清单.md` 追加一节: 该龟 普攻/被动/技一二三 各一条"看什么"(动作对不对/VFX对不对参考/数字飘法/无卡帧) + 美术项(立绘/技能特效是否到位).
