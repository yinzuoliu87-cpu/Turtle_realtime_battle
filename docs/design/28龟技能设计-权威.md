### R1 · 回合 → 秒 的换算
回合制的 `cd`/`*Turns` 是**回合**。实时版没有回合，**一律按各技能条目里写的秒数为准**（每条都写死了）。
换算基线（仅供理解，不是公式）：1 回合 ≈ 1 次普攻周期 ≈ **0.85 秒**（多数龟的攻速间隔）。

### R2 · 护盾有两类：永久盾 / 限时盾（**每处都已标明，别推测**）
实时版的护盾原语 `_grant_shield(单位, 数值, 时长)`：
- **省略时长 → 永久盾**（整场不过期，直到被打空）。当前 **18 处** 这样用：小龟龟盾、龟壳、石头嘲讽自盾、幽灵盾、既有盾等 —— **这批与回合制"永久护盾"一致，不是分歧。**
- **传入秒数 → 限时盾**（到期消失）。当前 **16 处**：天使祝福 5s、石头全队护盾 4s、竹叶自然恢复 4s、泡泡盾 4s、双头锤击 4s、寒冰团队盾 4s…
- 多来源护盾**叠加数值**；限时盾的到期时间取**更晚**的那个。护盾上限 = `最大生命 × SHIELD_CAP_MULT`。
- **真实伤害穿透护盾**（R5）。

> 回合制里标"永久"的盾，实时若改成了限时，本文档会在该技能条目**明写秒数并说明**；没写"限时"的就是永久盾。

### R3 · 多段合并
回合制常写"N 段共 X×ATK"。实时版若把它收成单段，**总伤害量不变**（例：竹叶一叶刃 3段×0.21A → 单段 0.63A 口径）。段数差异**不是数值分歧**。

### R4 · 龟能 = 冷却（同一件事）
实时版没有独立冷却计时器。`龟能` 固定速率充能，技能有龟能花费，攒满即放。
**"冷却秒数" = 龟能花费 × 0.075**（例：100 龟能 ≈ 7.5 秒）。全 UI 只出现"龟能"，不出现"CD/冷却"。
**龟能取值的唯一事实源 = `data/pets.json` 的 `energyCost`**（运行时 `_skill_cost` 优先读它；`scripts/systems/skill_energy.gd` 只是兜底表）。
> **2026-07-11 J1 收口**：此前三处 UI 各读各的 —— 战斗/图鉴用"pets 优先"，但**选龟界面只读 `SkillEnergy` 兜底表**，于是彩虹·棱镜护盾**界面显 70 而实战花 50**、彩虹·反射（表里没有）**界面完全不显龟能**。已把选龟界面（`TeamSelectScene`）也改成 pets 优先，并把 `SkillEnergy.SKILL_COST` 收成**恰好 84 个主动技**（＝全部 `skillPool[1..3].type`，删了 16 个普攻/已删技能的死条目，`shield` 70→50 对齐 pets，补 `rainbowReflect=110`）。
> **回归守卫**：`tests/verify_skill_energy.gd`（表 = 技能池且无冲突、普攻不占位）＋ `tools/tri_audit.py` A 段（每次 `run-tests.sh` 都跑）。

### R5 · 伤害 / 击飞 / 护盾 的统一口径
- **`N×ATK`** = 该龟当前攻击力 × N。**物理**吃护甲、**魔法**吃魔抗、**真实**穿减伤且**穿护盾**。
- **击飞滞空时长** = `2 × (KNOCK_VY × vy_mult) / |GRAVITY|`，其中 `KNOCK_VY=6.0`、`GRAVITY=-22.0` → **滞空秒数 = 0.545 × vy_mult**。文档里写"击飞 2.0 秒"即指滞空 2.0 秒（vy_mult=3.667）。
- **护盾**吸收伤害，被真实伤害穿透；到期消失（R2）。

# 附录D · 工程实装记录（2026-07-11 K/J 各轮 · 非技能数值，但影响后端与真机）

### D-1 · 立绘是 sprite-sheet，**最大边必须 ≤ 4096**（真机黑图根因）
- 每只战斗龟的 idle 立绘 = `data/pets.json` 的 `img`（`pets/<id>.png`）+ `sprite` 元数据 `{frames, frameW, frameH, duration}`。
- 加载端 `RealtimeBattle3DScene._sprite_dict_from`：`hframes = 宽/frameW`、`vframes = 高/frameH`，**支持任意网格**。帧序 row-major。**播放帧数 = `min(sprite.frames, hframes×vframes − 1)`**（`drop_last`，丢最后一帧对齐回合制）。
- **★2026-07-11**：11 张立绘曾是**超长横条**（`shell.png` 10000×500 / `fortune` 9000 / `headless` 8500 …），超过安卓 GPU 的 `GL_MAX_TEXTURE_SIZE`（ES3 规范下限 2048，真机常见 4096/8192）→ 纹理创建失败 → **采样即黑**（桌面 NVIDIA 上限 32768，端游一直正常）。已用 `tools/repack_sheets.py` 重排成网格（`shell` → 4000×1500，8列×3行），**像素逐帧逐字节不变**，同步把 `sprite.frames` 改成有效帧数。
- **接地渐隐 shader** 依赖行数：`_get_ground_fade_shader` 新增 `uniform vframes`，用 `fract(UV.y × vframes)` 取行内局部 y（旧版把整图当一行，网格化后除末行外底淡全错）。
- **守卫**：`tests/verify_sprite_sheets.gd`——全库贴图最大边 ≤ 4096、每龟有效帧数与重排前逐一相同、无半格。**新美术入库必须过这关。**

### D-2 · 循环 tween 必须 `bind_node`（否则目标被 free 后刷 `Infinite loop`）
