# 装备演示 / 测试配置登记表（EQDEMO）

> 每件需要专属演示的装备，在这里登记它的**完整测试配置** + **一键运行命令**。
> 以后要看某件装备效果 = 照抄命令、设好环境变量、实际跑起来看，不用每次重新搭场景。

## 机制：EQDEMO 测试台

战斗场景 `RealtimeBattle3DScene` 内置一套 env 门控的装备演示台（`REVIEW_DEMO` 评审模式的一个分支，正常游戏不受影响）。设 `EQDEMO_EQUIP=<装备id>` 即启用：

| 环境变量 | 作用 | 默认 |
|---|---|---|
| `EQDEMO_EQUIP` | 要演示的装备 id（如 `p2eq_034`）；**必填**，设了才进演示台 | — |
| `EQDEMO_STAR` | 装备星级 1/2/3 | `2` |
| `EQDEMO_CARRIER` | 携带者龟 id | `basic`（友方假人） |
| `EQDEMO_DUMMYHP` | 右侧敌方假人血量 | `5000` |

**默认场景布局**（写在 `_spawn_teams` 里）：
- **左 = 1 只友方假人携带者**：`no_move` + `no_basic` + 移速0 + 50万血，只负责持装备触发效果（不会自己乱跑/攻击，方便观察召唤/触发）。
- **右 = 2 只敌方假人**：`no_move` / 无技能 / 30 双抗 / 会掉血；**相距 500 码**（`_cx+60 + i*500`），方便看直线/位移/穿透类效果。
- 装备注入：`_inject_equipment` 对 `REVIEW_DEMO` 早返回，但 **EQDEMO 例外**（否则携带者拿不到装备）。

## 运行命令

Godot 可执行文件：`C:\Users\Louis\Desktop\Godot_v4.6.3-stable_win64.exe`
仓库：`c:\Users\Louis\Documents\GitHub\turtle-realtime-godot`

**开窗口看（推荐，墙钟卡时长，能看动作/位移）**——Git Bash：
```bash
timeout 42 env EQDEMO_EQUIP=p2eq_034 EQDEMO_STAR=2 \
  "/c/Users/Louis/Desktop/Godot_v4.6.3-stable_win64.exe" \
  --path "c:/Users/Louis/Documents/GitHub/turtle-realtime-godot" \
  res://scenes/RealtimeBattle3D.tscn --resolution 1280x720
```
> ⚠ 别用 `--quit-after N`（按帧算，窗口没锁 60fps 会提前关）；用 `timeout <秒>` 卡墙钟时长。
> ⚠ headless（`--headless`）只用来抓 parse / 运行时报错，**sim 时间不可靠**，看不到真实节奏。
