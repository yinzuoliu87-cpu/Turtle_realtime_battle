# 28龟 基础技能 (skillPool[0]) 重对账本

> 2026-06-28 用户："感觉直接移植基础技能有点问题" → 全28龟基础技能照原始 `skillPool[0]` 公式/类型/机制重对。
> 病根：旧 `BASIC_ATK [scale, hits]` 表是手填的，把"技能总倍率"误当"每段倍率"(×段数 = 数倍超伤)，且全成物理、丢了特殊机制。

## 架构 (RealtimeBattle3DScene.gd)
- `BASIC_ATK` 改成**每龟富 spec dict**：`phys/magic/true`=×ATK 总倍率(物/魔/真)、`hits`=视觉段、`def/mr/hp/selfhp/tcurhp`=加成项(进主类型)、`gold/critflat`=动态(财神/骰子)、`rider`=burn/atkdn/selfdef、`mech`=ninja/splash。
- `_do_basic(u,tgt,spec)` 数据驱动：算三类原始伤害(含加成)→`_mitigate`(暴击+减伤,与 _atk_dmg 同口径但吃绝对值)→分段打出→附带/机制。
- 闪电走专用 `_lightning_basic`(一道魔法+连锁2跳)；熔岩火山形态覆盖 spec。
