# 斗龟场 实时版 — 战斗设计（Botworld 式实时自走棋）

> 2026-06-27 立项。从回合制版 fork，**局外层全复用**，**战斗从回合制重做成实时**。本文是战斗设计 + 改造路线图。

## 一、Botworld 战斗模型（参照原型）
**实时「定位型自走棋」**（比 TFT 动作化）：
1. **战前布置**：双方同时把单位放进竞技场（位置），看不到对方 → 站位是核心。
2. **战斗中全自动**：单位自己跑（**索敌=最近敌人**，近战追/远程风筝/坦克顶前）+ 到射程按攻速自动普攻。**玩家不直接控单位移动**。
3. **玩家在战斗中 = 主动放技能/武器**：自己决定**何时放 + 放哪**（瞄准落点），技能要**充能/读条**，还能引敌进地图陷阱。
4. **7 角色决定 AI 行为**：追击/闪避/格斗/坦克/狙击/泼溅/辅助。
5. **胜负 = 消灭对方全队**。

### Botworld bot 技能参考（设计龟技能的模板）
每 bot = 普攻 + 1 特色技能；按角色：
- 坦克: 反伤回血/推+眩晕/正面减伤/控场连招/极限吸伤
- 泼溅(AoE): 多目标/击退/自爆溅射/机动AoE核/眩晕
- 狙击(远程): 稳定DPS/玻璃炮(要敌静止)/高爆单发
- 追击(冲脸): 最高伤脆皮/最高DPS免控突进/高血抛敌
- 闪避(风筝): 贴脸AoE/近距全AoE/强控/隐身
- 格斗: 1v1眩晕锁/超高伤需配合/劫持敌方单位
- 辅助: 治疗+buff/伤害buff+眩晕/RNG技能

## 二、地基决策（用户 2026-06-27 拍板）
- **2D 俯视自由走位**（不是 3D）—— 复用现有 2D 像素美术；龟在 2D 俯视战场自由移动。Botworld 是 3D，但 3D 只是视觉、玩法在 2D 俯视照样跑，且 3D 要全废 2D 美术重做模型 → 不值。
- **视角 = 元气骑士式俯视**：相机从上方斜看竞技场，龟用**现有侧面精灵、面朝移动方向**在场上跑（像 元气骑士/荒野乱斗）。最省美术、侧面图直接用。
- **自由走位（Botworld 式真实时）**—— 龟自由跑/追/风筝/躲；移动 + 寻路 + 实时索敌**从零做**。
- **单场俯视竞技场、队 vs 队、灭对方全队 = 胜**（去双路+龟蛋）。

## 三、新战斗模型（⚠ 全新设计，不是"复用回合制"— 此前规划夸大了复用）
**诚实**：实时战斗是**新系统**。回合制只有"伤害算术"那点能复用；下面这些**回合制根本没有、全是新设计**：
- **新属性（每龟都得设计赋值）**：攻速(攻击间隔 s) · 移速 · 攻击射程 · 近战/远程。回合制龟没有这些 → 按 龟类型/角色 设计 + 调。
- **攻击模型**：近战(移动到接触→攻击) vs 远程(射程外发投射物)，按龟类型分。
- **移动 AI（按角色）**：近战追最近敌 / 远程风筝拉距离 / 坦克顶前 / 辅助跟队 / AoE 躲后。
- **技能模型**：龟能随时间回(`_energy += regen×delta`) → 满了 **玩家点放 + 瞄落点(AoE)**（Botworld 主动乐趣）或默认自动放。
- **入场站位** = 复用 6 格定位网格 → 战场初始坐标。
- **战斗结构（已定 2026-06-27）** = **单场俯视竞技场、队 vs 队、灭对方全队 = 胜**。去掉双路 + 龟蛋（对齐 Botworld 单场，最简洁）。

### 🔴 架构级盲点（不先定，战斗做出来会跟整个局外 meta 对不上）
1. **实时版还用不用 V2 异步 PvP 那套 meta？** 实时版是从 V2（异步 ghost 生命赛季）fork 的，但实时战斗 + 玩家主动放技能 跟"异步打 ghost 快照"**根本冲突**：①ghost 是死快照（没玩家操作），你实时主动放技能 vs 纯 AI 托管 ghost = **不对等**；②排行榜原按"击杀蛋数"排，**但我们删了龟蛋 → 排行指标没了**；③防作弊靠后端复算，**实时战斗确定性回放极难**。→ 实时版要么重新定义异步怎么打/排行排什么，要么换个模式结构。
2. **战斗要不要确定性/可回放？** 异步公平 + 防作弊需要可复算；实时（帧率/物理/随机）确定性很难做。定了它，才知道随机/物理能不能随便用。
3. **玩家主动性 vs 异步对手** 你实时放技能（主动），ghost 只能 AI 自动 → 非对称体验，怎么处理。

## 时间制转换（⚠ "回合"概念彻底删除 — 全部计时改秒/实时）
用户点出（2026-06-27，以冰寒为例）：**冰寒不再有"持续回合"，只有"持续几秒"。这适用于游戏里每一个计时量** —— "回合"是回合制的心跳，实时下心跳是时钟（秒/delta），所有 per-turn 量必须重表达为 per-second 并**重新调值**：

| 原(回合制) | 实时 |
|---|---|
| 控制时长 冰寒/冻结·眩晕·嘲讽·恐惧·混乱 "N 回合" | "N 秒" |
| DoT 灼烧/中毒/流血/诅咒：回合末结算、"X 伤/回合 × N 回合" | **每秒 tick**、"X 伤/秒 × N 秒" |
| buff/debuff 时长 "N 回合" | "N 秒" |
| 龟能回复 "每回合 +max×40%(turn≥2)" | "每秒 +Z" |
| 技能 CD（cd 字段废弃，每 round tick） | **启用为实时秒 CD** |
| 决胜局 "turn>30 → 怒气/疲惫" | "战斗时间 > T 秒" |
| 环境事件 turn 3/6/9/12（若留） | 按秒 |

**转换原则（不是机械相等！）**：不是 "2 回合 = 2 秒" 硬换。每个时长按**实时手感重新设计**（冰寒太长=stunlock 不好玩 / 太短=没用），先给起步值，再 **F5 playtest 调**。

**诚实**：这些"几秒"的**具体数值 = 设计 + 调试任务，纸面定不准**（实时手感靠 playtest）。前面 plan 标了（盲点 #14 + 再平衡 #12 + 阶段3），但 **"标了" ≠ "定好了"** —— 框架无洞，但每个计时值我明确留作 **draft + F5 调**，不假装规划好了。

**诚实复用边界**：✅ 伤害公式(atk×系数/暴击/护甲) + 效果应用(DoT/控/buff，但计时改"秒") + 龟/装备数据(名/美术) + 局外层。 ❌ 重写：战斗驱动 + 索敌 + 移动 + 攻击/技能时序；很多回合制专属技能（"本回合行动两次"/"回合末"）**没法映射、要重设计**。
