## 2. 技术架构

### 2.1 为什么不用等距 / 相机 yaw
- 游戏是 **2.5D 透视**（billboard 立绘 + 透视相机）。等距=换正交相机+菱形世界+全部 VFX/坐标/立绘重调+重画美术，**纯负担无收益**。
- "菱形地砖"的观感 = **相机俯视角**下的正方形自然呈现，**不需要**把世界转 45°。

### 2.2 tile 地面渲染：MultiMesh（而非 GridMap）
- **每种 tile type 一个 `MultiMeshInstance3D`**（如 grass/water/stone/sand…），每格是一个 quad/薄 box，按 grid 坐标 + 该格 height 定位。GPU 实例化，几百格无压力。
- **选 MultiMesh 不选 Godot GridMap 的原因**：我们要**精确 per-tile 高度**（凹水池 / 抬高石台），GridMap 的 cell 高度是分层的、不顺手；且 MultiMesh 数据走 `map.json` 更**可移植 + 能做局内编辑器**，契合本项目"全自研"的风格。
- （备选：若你更想用 Godot 编辑器里画，GridMap+MeshLibrary 也能做，编辑器白捡；但高度和 json 移植性差。默认走 MultiMesh。）

### 2.3 数据格式 `data/maps/<name>.json`
```json
{
  "w": 40, "h": 24, "tile": 24,
  "types": ["grass","water","stone","sand","void"],
  "grid": [[0,0,1,1,...], ...],      // 每格 type 索引
  "height": [[0,0,-0.6,-0.6,...], ...]// 每格高度(米), 水=负, 石=正
}
```
- 载入器：读 json → 按 type 分桶填各 MultiMesh + 设每格 transform(含 height)。
- 一张手工 `arena.json` 对齐现有 `ARENA` 边界：中央凹水池 + 两端抬高石台(龟蛋位) + 四周主地面。

### 2.4 tile 类型（起步 5 种，可扩）
| type | height | 材质(暗调) | 备注 |
|---|---|---|---|
| grass 主地面 | 0 | 暗蓝紫 + 斜网格线 | 铺满的底 |
| water 水 | -0.6 | 青色发光 + 滚动波纹 + 边缘亮边 | 中央池 |
| stone 石台 | +0.4 | 暗石板 + 网格 | 龟蛋位/边角 |
| sand 浅滩 | -0.1 | 暗金沙 | 水陆过渡 |
| void 空 | — | 不渲染 | 挖洞/边界 |

## 3. 分阶段落地计划（每阶段一板 · 纯色占位先跑通 · 最后才换皮）

**阶段0 · 相机定档 + tile 渲染原型**
- 相机俯角/ FOV 给 3 档 F5 挑，定一档。定后重调 VFX 贴地偏移/血条飘字/立绘高度。
- MultiMesh 方块地面原型：硬编一张测试 map（**纯色**，无美术），跑通"数据→几何→相机下读得对"。

**阶段1 · tile 数据 + 地图载入**
- 定 `map.json` 格式 + 载入器 + 5 种 type。
- 手工 `arena.json`（中央凹水池+两端石台+四周地面），对齐 ARENA。
- 水面 shader（滚动波纹+边缘亮边）；height 生效（水凹/石凸）。

**阶段2 · 暗色调 + 斜网格贴图**
- 各 type 材质套**暗深海夜调色板**（下 §4，硬锁）。
- 斜向网格线 + 格内明度斑驳**烘进 tile 贴图** → 菱形地砖外观。
- 暗环境光 + 中央柔光压低（保证技能特效跳出来）。

**阶段3 · 局内刷子编辑器**
- 调试态（env/快捷键开）：选 type+height，鼠标在网格上刷，保存/载入 `map.json`，撤销。→ 能刷多张地图。

**阶段4 · 植被/框边/氛围**
- 装饰层：沿边框/四角高密度撒发光水草/珊瑚/花（程序化 + 可编辑器点放），空地<40%，中央+双路+基地留空不挡视线。
- 暗色高立绘框边聚焦中心；相机跟随漂浮辉光粒子层。

**阶段5 · 换素材（PixelLab · 暗调）**
- tile 贴图/水草/珊瑚/石台/水面…各 3-4 变体，全锁下面的调色板。
