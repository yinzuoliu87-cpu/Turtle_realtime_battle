## A · GPU / 渲染（黑屏最可能出这）
| # | 疑因 | 状态 | 依据 / 处理 |
|---|---|---|---|
| **A1** | 时停 2 个屏幕纹理 shader | ✅本轮修 | `gl_compatibility`(ES) 下读屏幕纹理需拷帧缓冲, 很多安卓 GL 驱动处理不好→**整屏黑**。且只在【沙漏时停触发】时走→正对"偶尔"。→ 改成**不读屏的普通压暗覆盖层** |
| **A2** | 贴图超 `GL_MAX_TEXTURE_SIZE`(真机常见4096) | ✅已修 | 11 张长条立绘重排成网格; 复查全库 **0 张 >4096** |
| **A3** | 全库无损贴图 → 显存 945MB → GPU 分配失败黑屏 | ⚠需真机(E-1) | 低端机 OOM 风险。开 ETC2 压缩显存÷4 → 待用户拍板 E-1 |
| **A4** | SubViewport MSAA 在移动端 | ✅本轮修(移动端关) | 移动端 SubViewport + MSAA 部分 GPU 有问题 → 移动端默认关 MSAA |
| **A5** | 其它自定义 shader 移动端兼容 | ✅已审 | 全部 hint_screen_texture(3处: 1注释+2真用)真用的2处均已 desktop-gate; 其余 spatial/canvas shader 无 depth texture/textureLod/dFdx 等 ES 麻烦语法 |
| **A6** | SubViewportContainer 合成在移动端 | ⚠需真机 | 注释说 GL Compat 才用 SubViewport; 合成层若失败→黑 |
