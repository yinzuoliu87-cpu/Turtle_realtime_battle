## 七、⚠️ 已发现待决：TeamSelect 残留回合制框架（拍板项）
**现状**：TeamSelect 是回合制版 1:1 复用，带两处回合制框架，**实时战斗完全不读 → 玩家做的选择静默无效**（违"不准占位/专业"标准）：
1. **6格前排/后排定位网格**：实时 `_spawn_teams` 只把 3 龟纵向铺开，**完全不读 left_slots/前后排** → 定位选择无效。(实时本是自由走位 3v3)
2. **"技能 选3" loadout**：实时 `_cast_active` 每龟**硬编 1 个主动技(候选1)**，不读 loadout → 选技无效。(设计草案是 4选1 选 1 主动技)

**为何未自行改**：① 6格→3格 / 去前后排 是对精心 1:1 复杂场景的不可逆大改；② 技能模型(4选1 选1 vs 保留选3)+让选择真驱动战斗(需实装候选2-4 × 28龟 大工程) = 核心玩法设计分叉。两者均为用户保留的**拍板** → 已向用户提问，定了再做。
