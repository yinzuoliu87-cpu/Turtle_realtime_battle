# 双路 + 小将 + 龟蛋 战斗设计（用户 2026-07-01 口述 → 下一轮规划用）

> 用途：这是"把双路/小将真正接进 3D 战斗"那个大方向的**设计源**。下一轮规划时据此 + **回合制版详细逻辑** 制定完整实施计划。当前轮（装备/背包/合成）先不动战斗。

## 一、用户原话要点（忠实记录）
- **双路 + 小将 就放在 3D 大战场里**（不是单独场景，用现有 `RealtimeBattle3DScene`）。
- **地图 = 大型竞技场**（arena）。
- **龟蛋**：在**左右两侧基地内**，被**"全息围栏"**围住。
  - 围栏内的龟蛋**可以吃到 AoE 伤害 或 增益**（围栏不挡 AoE / buff）。
  - 围栏在时，龟蛋**不会被主动索敌**（敌方单位不会自动寻路/锁定它）。
- **战斗开始前**：有一个**放置区域**让玩家摆放自己的 **3 只龟**。放好 3 只 → 战斗开始。
- **一方团灭后**：该方**围栏消失 10 秒** → 龟蛋被**自由攻击 10 秒** → **上半场结束**。
- **详细逻辑** 要**参照回合制版**补全。

## 二、我的理解（待下一轮细化 + 跟用户确认几处）
- 玩法结构 ≈ MOBA 竞技场：放置 3 龟 → 龟 + 小将混战 → 团灭对方 → 10 秒破蛋窗口 → 半场结束。
- **"上半场"对应双路**：上路 = 上半场、下路 = 下半场；`egg_hp` 跨路累积（已在 `phase2_duallane.gd` / `GameState.damage_egg`），谁的蛋先破谁输。
- **围栏 = 龟蛋保护罩**：平时不可主动索敌（安全，逼你先清场），团灭后 10 秒才可自由破蛋（AoE / 增益 平时就能碰到蛋 → 有策略空间）。
- ❓待确认：① "3大战场" = 3D 大战场 还是 3v3？② 放置区域是双方各自放、还是只玩家放（对手 ghost 自动布阵）？③ 上下半场之间是否重置阵容/小将？④ 龟蛋破了立刻结束还是"自由攻击满10秒才算上半场结束"。
