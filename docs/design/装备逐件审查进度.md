## 2026-07-19 本轮评审改动摘要

### 修掉的真 bug
- **017 不沉之锚**：`_knock_immune` 是只写不读的死标记，「免疫击飞」从未生效。补上守卫后又发现全场 **5 处**直接设 `airborne` 绕过该判定（无头镰刀 / 熔岩浪 / 熔岩砸地 / 通用 `_knock_up` / 029 冰道），一并补齐。
- **019 海葵药膏**：海葵层的「治疗与护盾强度 +8/9/10%」只在自己函数内当局部倍率用，从未写进 `u.heal_amp/shield_amp` —— 对其他治疗来源和所有护盾一律无效。
- **035 黄铜齿轮**：`gears` 字段全项目无人 +1，`_settle_gears()` 恒结算 0，而头像徽章一直显示「0」。改为显示本局累计产币数。
- **059 沙漏**：时停期间全局 `_t` 冻结，但 active 携带者仍在行动 —— 它身上所有时间戳型状态（眩晕/嘲讽/减速/buff 到期）永远不到期，被眩晕就会晕满整个时停（⭐3 达 30 秒）。
- **彩虹龟·棱镜护盾**（评审外顺查）：type 是泛用 `"shield"`，被路由到通用实现，2026-07-13 专门写的七彩棱镜版从未被调用，且丢了 4 秒时长。
- **换路特效残留**：VFX 的 `queue_free` 挂在 tween 末尾回调上，而换路时 `tw.kill()` 一杀该回调永不执行 → 精灵跨场残留。改为「建场快照 + 换路兜底扫场」治本（实测每次换路扫出 35–38 个遗留节点）。

### 卡死预防（同 053 那次冻死的同族问题）
- Godot 对 Dictionary 的 `==` / `!=` / `in` / `.has()` / `.find()` / `.erase()` 都是**深比较**，而单位字典互相引用成环 → 递归哈希卡死。
- 三轮共修 **40 处**。前两轮用「变量名白名单」写正则，连着漏网三次；第三轮改成**不预设名字**抓所有「裸标识符 ==/!= 裸标识符」再人工判类型才收干净。
- 已备好引用语义 helper：`_arr_has_unit` / `_arr_erase_unit`（战斗场景）、`_idx_of` / `_arr_has_ref`（phase2_equip_runtime）。

### 弹道收口
- `_spawn_eq_bolt` 家族（048/049/050/052/056）原用 `billboard + rotation.z`，**billboard 每帧重置朝向、roll 被吃掉**，弹头永远水平；`0.55` 还是个假的俯视前缩系数。
- 全部迁到 `wisp_dir`（相机 basis × 屏幕投影 roll）；旋转弹（056 飞镖）走 `card_spin`。
- 新增 `wisp_off` 参数修正纹理朝向：`wisp_dir` 默认假设纹理朝前是 +Y，而 `bullet.png`/`crossbow-bolt.png`/`bamboo-arrow.png` 都是横向贴图（+X 朝前），需传 `PI/2`。

### 文案与代码对齐
- **7 件**还在用回合制的「列 / 排 / 格」说法（024/030/034/039/043/051/058），实际都是「朝某方向拉直线 + 半宽判定」。
- 「局」与「战场」用词修正 5 件：一局含上路/下路/终极**多个战场**，这些状态是**每场战斗**重置，不是每局。
- 034 大熊那一整套冲击波机制（熊掌攒层→满 2 层射程扩到 600 码→1.5×ATK+击飞+拉回）原描述一字未提。
- 056 靶子「短时间」实为**永不过期**（`_t + 99999`）。
