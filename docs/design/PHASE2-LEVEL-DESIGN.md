## 1. 核心概念 — 两种「等级」并存, 别混
- **乌龟持久等级** (已有, per-pet, 1-10): 存档里每只龟的等级, 影响龟 base 属性. **保留不动.**
- **局内等级** (新, per-player, 1-10): 像 TFT 玩家等级, **每局重置**, 靠局内 XP 升. 绑定龟蛋HP + 商店概率档.
  - 双方各自一个局内等级 (PvP 各自升).

## 3. 升级机制 (XP, 仿 TFT) — 数值占位待定
- **被动XP**: 每大回合自动 +`PASSIVE_XP`(占位 2).
- **买经验**: 花深海币买 XP (占位 4币 = 4XP), 玩家主动点.
- **每级XP阈值** (升到下一级需累计, 占位曲线):
  | 当前级 | 升级所需XP |
  |---|---|
  | 1→2 | 2 |
  | 2→3 | 6 |
  | 3→4 | 10 |
  | 4→5 | 16 |
  | 5→6 | 24 |
  | 6→7 | 36 |
  | 7→8 | 52 |
  | 8→9 | 72 |
  | 9→10 | 96 |
  (满级10不再升. 曲线要按"一局约多少回合能到8-10级"实测调.)

## 4. ⚠ 待用户拍板 (规划阶段的关键决策)
1. **龟蛋HP随升级怎么变?**
   - **A 强化(推荐, TFT味)**: 蛋 max=1000+50×级; 升级时 max+50 **且 current+50**(强化基地), 累计伤害保留 → 升级=补蛋血, 防守有价值.
   - **B 开局锁定**: 蛋HP开局按当时等级定死, 之后升级只影响商店, 不再动蛋.
2. **怎么获得等级?**
   - 纯被动(每回合自动升) / 纯买经验(花币) / **两者都有(推荐, 同TFT)**.
3. **等级要不要也带动别的?** 用户只说绑蛋+概率. 默认**不动龟属性**.
   - 可选: 让**补位小将等级 = 局内等级**(小将随局变强, 合理); 龟属性默认不绑.
4. **敌方(AI)怎么升级?** 默认: AI 跟玩家同节奏自动升(或按回合数推等级), 保证蛋+商店对等.

## 5. 代码落地 (确认决策后)
- `GameState`: `dual_level{left,right}` + `dual_xp{left,right}`, 每局重置; `add_xp(side,n)` / `try_level_up` / `buy_xp(side)`.
- `Phase2Config`: `LEVEL_XP_THRESHOLDS[]` + `PASSIVE_XP` + `BUY_XP_AMOUNT/COST` (占位常量集中).

- UI: 大地图 + 战斗内 显「等级 N + XP条 + 买经验按钮」(像TFT左下角).
- PvP: 各自等级走权威+快照同步(已有 side_controllers/snapshot 框架).
