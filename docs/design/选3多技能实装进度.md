## 〇、龟能/冷却模型 (2026-06-28 用户纠错 → 对齐 Botworld)
**用户两问纠错**: ①"龟能充能=冷却时间吗?" ②"龟盾和龟派气波怎么可能冷却一样?"。查 Botworld wiki 实证: **大招是【充能条】, 靠"对敌输出伤害"攒满, 非固定计时冷却**(*"Bots charge up an Ultimate if they deal enough damage"*); 普通技各有**独立冷却**(AI升级=更快冷却)。→ 我原模型双错: 龟能按**固定时间**回(≈7s, 等于冷却)+ 所有技**同一阈值**(龟盾=气波 同频, 荒谬)。skill 数据 `cd` 字段大多 0 也印证回合制源没建这套。
**改后 (Botworld 式)**:
- **龟能靠输出攒**: `_basic_attack` 打出的伤害 ×`ENERGY_PER_DMG`(0.5) → 龟能; 另有 `ENERGY_TRICKLE`(4/s)被动微回防卡死; 上限 `ENERGY_CAP`(200)。高DPS龟大招更勤, 坦克更慢 = 真 Botworld。
- **各技独立 cost (∝强度)**: `SKILL_COST` 表 (龟盾/小盾治~70 · 普通伤害~95 · 全体/多段~120-140 · 变身/梭哈/大招~150-170)。`_cast_active` 攒够【当前轮转技】的 cost 才放、花掉对应龟能 → **龟盾便宜常放 / 气波·弹幕·大招 贵且少放**。龟能条显"当前/下一技 cost"进度。
- 数值首版 F5 调 (ENERGY_PER_DMG / TRICKLE / 各 cost)。

## 一、框架 (RealtimeBattle3DScene.gd)
- **轮转模型**: 每龟读自己 loadout(选3) → `active_skills` 轮转表(`_resolve_active_skills`): `physical`/`magic` 是普攻(自动攻击)排除, 其余按选中顺序进表。
- **触发**: 龟能(输出攒) ≥ 当前轮转技 cost → `_cast_active` 放(`_cast_skill`)、花龟能、轮转推进。未实装/梭哈已用的**跳过**(不空放), 不放假招(无 placeholder)。
- **分派**: `_cast_skill` 用 `_IMPL_SKILLS` 集判实装 → juice(预备/震屏)+VFX → `_do_skill` 按**技能 type**(非龟 id)分派。
- **数据驱动伤害**: `_sk_dmg(u, tgt, opts)` 系数取自 `pets.json` 各技 `detail` 公式 `{N/M/T:...}`(N=物理 M=魔法 T=真实, ×caster ATK; hp/mr=×caster maxHp/MR 附加; hits=视觉段数封顶8防刷屏, 总伤不变; aoe=全体; rider=burn/stun/slow/curse/atkdn/mrdn)。
- **左右队 loadout**: 左队(玩家)读 `GameState.loadouts`; 右队(ghost/bot)用 `defaultSkills`。
