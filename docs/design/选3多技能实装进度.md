## 〇、放招/冷却模型 (2026-06-28 用户实测纠错 → 对齐 Botworld 真相)
**两轮纠错**: 用户先问①"龟能充能=冷却吗" ②"龟盾和气波怎能同冷却"。我第一轮**靠网页搜索误判成"靠输出伤害攒大招"(错!)** —— 病根: WebSearch 给的是二手攻略再复述、每次不一样、还跟 wiki "missing data" 冲突。用户怒指"每次查都不一样到底怎么搞", 然后**自己进游戏实测**(沙蝎): "**充能条不断走→满放大招→再走**", 且数据写死 躲避CD5s/大招CD13s/普攻0.5s。
**Botworld 真相(实测定案)**: **逐技各自一个【固定时间冷却】(按秒, 与打没打中无关); "大招"=冷却很长的技, 那个充能条 = 该技冷却进度条。** 没有"能量/伤害充能"这回事。
**改后(最终)**:
- 逐技独立冷却走时间(`skill_cd` 每技剩余秒, `_regen_energy` 每帧扣 delta); 冷却好且有目标 → 放【就绪里强度最高的】那个(大招好了优先放、小技填空档), 重置该技冷却 + `SKILL_GCD`(0.4s)防同帧连爆。
- 冷却秒数 = `SKILL_COST`(∝强度) × `SKILL_CD_FACTOR`(0.075): 龟盾~5s · 普通~7s · 弹幕~10s · 大招~13s(对齐沙蝎 dodge5/ult13)。→ **龟盾CD短常放 / 气波·弹幕·大招CD长少放**。
- 充能条(en_fill)显"最快要好的技"的冷却进度。数值首版 F5 调(`SKILL_CD_FACTOR` + 各 cost)。
- **教训**: 别拿 WebSearch 二手复述当事实(会翻来覆去); 拿不准的游戏机制 → 用户实测/截图 才是 ground truth。

## 一、框架 (RealtimeBattle3DScene.gd)
- **轮转模型**: 每龟读自己 loadout(选3) → `active_skills` 轮转表(`_resolve_active_skills`): `physical`/`magic` 是普攻(自动攻击)排除, 其余按选中顺序进表。
- **触发**: 龟能(输出攒) ≥ 当前轮转技 cost → `_cast_active` 放(`_cast_skill`)、花龟能、轮转推进。未实装/梭哈已用的**跳过**(不空放), 不放假招(无 placeholder)。
- **分派**: `_cast_skill` 用 `_IMPL_SKILLS` 集判实装 → juice(预备/震屏)+VFX → `_do_skill` 按**技能 type**(非龟 id)分派。
- **数据驱动伤害**: `_sk_dmg(u, tgt, opts)` 系数取自 `pets.json` 各技 `detail` 公式 `{N/M/T:...}`(N=物理 M=魔法 T=真实, ×caster ATK; hp/mr=×caster maxHp/MR 附加; hits=视觉段数封顶8防刷屏, 总伤不变; aoe=全体; rider=burn/stun/slow/curse/atkdn/mrdn)。
- **左右队 loadout**: 左队(玩家)读 `GameState.loadouts`; 右队(ghost/bot)用 `defaultSkills`。
