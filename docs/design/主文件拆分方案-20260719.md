### A. 沙漏时停 `_ts_*` — 推荐优先
- **18 个函数 / 320 行**
- 外部函数依赖 **5 个**：`_eq_grant_energy` `_float_text` `_has_energy_system` `_skill_ring` `_world_pos`
- 外部成员变量 **6 个**：`_cam` `_follow_vfx` `_sim_tweens` `_t` `_units` `_world`
- **阻力**：`_ts_active` / `_ts_remaining` 被主循环 `_process` 直接读写（每帧判断是否进入时停分支）。拆出去后需要主循环通过接口查询，属**双向耦合**。
- 备注：体检报告估计此块 817 行，实测按 `_ts_*` 函数统计只有 320 行（3291–4107 那个区间里混着别的函数）。

### B. 调试场编辑器 `_edit_*`
- **40 个函数 / 631 行**，且是 **DEV-only 代码躺在出货战斗场里**
- 外部依赖仅 5 个：`_units` `_data_by_id()` `_make_unit()` `_cam` `_spool`
- **阻力**：`_unhandled_input` 里与统计面板 / 双路放置 / 触屏 pinch 的输入分发混在一起，**要先抽一层输入路由**才能整块搬走。
- 收益最大（行数 + 概念纯度），但改动面也最大。

### C. 战中统计浮层 — ✅ 已完成（2026-07-19）
抽出 6 个函数 + 4 个成员变量 + 6 个常量 → `DmgStatsPanel`，主文件 −165 行。
新类对外只有 `setup(ui_layer, units_of)` / `toggle()` / `render()`，不认识 `_units`／`_world`／战斗状态。

**纠正一处此前的归类错误**：`_stats_column` 属于**结算统计表**（`_build_stats_panel`，7 列表格），
不是战中浮层——早先的依赖扫描把两者混在了一起。且它用 `await get_tree()` 需要 Node 上下文，
与 RefCounted 类不兼容。故本次只抽战中浮层，结算表整体未动。

**结算统计表仍可作为后续候选**（`_build_stats_panel` / `_center_panel_deferred` / `_stats_column`，约 90 行），
但需以 Node 形态承载；且注释写明其样式是"1:1 照抄回合制 BattleEndScene._stats_table"，
即同样的表格构建代码在项目里仍有两份，抽出可跨版本复用。

- `Sprite3D.new()` 全文 335 次，其中约 231 处是同一套 10 行模板

- **暂缓理由**：它最主要的价值——防止"tween 被 kill → `queue_free` 回调不执行 → 特效跨场泄漏"——**已被 2026-07-19 的"建场快照 + 换路兜底扫场"从根上解决**（实测每次换路扫掉 35–38 个遗留节点）。剩下只是行数收益，而 231 处写法各异，在 22k 行文件里盲改的风险与收益不对等，且无法逐一眼验 28 只龟的技能表现。

## 拆分作业的两条硬教训（2026-07-19 踩过）

1. **切函数块的边界扫描，遇到任何列首注释就必须停**。第一版只在"分隔符注释（`# ===` / `# ───` / `# ═══`）"处停，遇到普通 `#` 说明注释会继续往下吃，结果把紧跟其后的 `_reticle_flash` 连注释带函数头一起卷走，函数体残留在上一个函数里 → `Parse Error: already a variable named "r"`。
   **护栏**：①每个块内 `func` 行必须恰好 1 个（它自己）②块之间不得重叠 ③拆完做**函数总数对账**（原 N = 主文件 + 新文件）。

2. **新建带 `class_name` 的脚本，必须先跑一次 `--headless --import`** 注册全局类，否则运行期报 `Identifier "XXX" not declared`（与新增 PNG 需要导入是同一类问题）。
