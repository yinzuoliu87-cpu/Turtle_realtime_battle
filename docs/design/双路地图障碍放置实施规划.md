## 决策记录（用户 2026-07-05 锁定 + 自主）
- **地图布局 = B「中央大礁 + 两侧错位掩体墙」**（用户选，符合设计稿）。
- **放大倍率 = 1.4×**（用户选）：ARENA 1140×520 → 1596×728（保中心/宽高比）；硬编码偏移同步 ×1.4；相机 pos ×1.4 拉远，角度不变。
- **三战场同一张地图**（用户选，先跑通）。
- **龟蛋 = 单颗蛋 + 3帧 idle 动画**（关键发现）：`pets/egg.png` 是 237×80 = **3帧 79×80 的单蛋 idle 动画帧**，realtime `_egg_sprite_dict` @1306 却当 `frames:1` 用 → 把 3 帧并排显示成"3蛋一窝"(bug)。修：hframes=3/frames=3/loop/fps~5 → 单蛋会动。**死亡碎裂帧两版都没有 → PixelLab 新做**。
- **navmesh 用 2D（NavigationServer2D + NavigationRegion2D）不用 3D**：战斗 sim 全在 2D 像素空间(ARENA/pos:Vector2)，3D 只是 billboard 渲染。障碍 footprint、寻路、分离都在 2D；渲染经 `_world_pos` 转 2.5D。（设计文档原写 3D，按 sim 实际改 2D）
- **障碍 footprint 用椭圆**（圆心+半径xy）而非精确多边形：够 navmesh 挖洞 + 分离外扩，简单稳。
- **障碍只挡移动**：弹道/AoE/索敌全不看障碍（不碰 `_nearest_enemy`/弹道/技能）。
- **凤凰龟（不停步攻击）默认也绕障走地面**，不飞过障碍（待用户确认）。
- **放置区 = 你半场一块矩形**，自由拖拽落点 clamp 在区内 + 避开障碍/基地 footprint（自主，可调）。

## Phase M1 — 场地放大 + 相机拉远（先不加障碍，只放大跑通）
**目标**：放大后现有双路战斗照常跑，单位在更大场里散开、不出界、相机框住全场。
1. `ARENA` → `Rect2(70,110,1596,728)`（1.4×）。**grep 全部 `ARENA` 引用逐个核**：多数用 `.position/.size/.end` 自适应；硬编码偏移 `_cx±300`→`±420`、lane `±110`→`±154`、蛋 `±70`→`±98`。
2. 相机拉远：`pos`→`(0, 18.9, 25.5)`(×1.4)，`look_at` 仍指 ARENA 中心的地面点，重算 `_cam_base`；fov 保持 50。
3. 地面网格尺寸随 ARENA×WS 放大（M2 会重做地面，这里先把尺寸接对）。
4. **验**：`DUALLANE=1 --headless` 跑通不崩；再非 headless 跑 + `ui.ps1` 截图：单位散开、框全场、无出界。逐处坐标改动 grep 确认，HUD 是 screen-space 不受影响。
- 风险：坐标类改动面广 → 每改一处 grep 复核；一次只放大不加障碍，隔离风险。1 commit。

## Phase M2 — 深海礁盘 地面 + 海床贴图（PixelLab）
1. 先定位地面节点（_ready 里的 MeshInstance3D/shader 地面）。
2. PixelLab 生成**可平铺海床瓦片** 512×512（深蓝绿砂石+珊瑚/贝壳点缀，边缘无缝）→ `assets/sprites/map/seabed_reef.png`。

## Phase M3 — 障碍物 + 基地围栏（billboard + 2D footprint）
1. **布局**（对称，2D ARENA 坐标）：中央大障碍 ×1（沉船/大礁，椭圆 footprint ~350×200px 居中）；两侧掩体墙 ×2（上下通道感）；两端基地平台（蛋位已有）。存 `_obstacles = [{center:Vector2, rx, ry, spr}]`。
2. PixelLab 生成：大礁石立绘、岩脊/墙立绘、**基地穹顶围栏**（半透明发光罩，罩蛋）→ `assets/sprites/map/`。
3. 障碍 = billboard `Sprite3D`（`_world_pos(center, h)` 浮起）；footprint 存 2D 椭圆。
4. 围栏 = 半透明发光穹顶 billboard 罩蛋，接现有 `_egg_fence`：团灭掉栏时淡出。
5. **验**：截图核对障碍立场中、穹顶罩蛋、对称、有掩体感。1 commit。

## Phase M4 — navmesh 2D 避障寻路（核心，最谨慎）
1. 建 `NavigationRegion2D` + `NavigationPolygon`：外轮廓=ARENA 矩形，挖洞=每个障碍 footprint 椭圆近似多边形（+单位半径 margin 外扩）。`bake` 一次（障碍静态）。
2. **接入移动**（`_do_move`）：目标点与当前点之间被障碍挡时（或每单位 ~0.5s 缓存重算），`NavigationServer2D.map_get_path(map, pos, tgt)` 得路点；单位朝**下一路点**走（替换直奔目标向量），到点阈值放宽切下一点；保留 `_apply_separation_pass` 绕友微调。
   - **兜底**：无 path / navmesh 未 ready → 直奔目标（现行为），绝不崩。
   - 攻击期定身/前摇豁免照旧（navmesh 只管 move 段）。
3. 障碍**只挡移动**：不碰索敌/弹道/AoE。
4. **验**：`DUALLANE` 跑 + `XDBG` 打印 path 点数；非 headless 截图/连拍看单位绕过中央大障碍到敌方、不穿模不卡死。1 commit。
- 风险最高：path 跟随 + 分离可能抖 → path 点当"移动目标"、分离当微调、到点阈值放宽、path 每 ~0.5s 重算(性能)。

## Phase P — 场内自由放置阶段（依赖地图）
1. 进每战场先 `_dl_state="place"`：我方该路单位出现在**自己半场放置区**(ARENA 我方半侧矩形)，敌方自动布阵。
2. 拖拽：点选单位→拖到位置（复用 `_edit_unit_at_screen` 屏→2D反投影 @9127 一带）；落点 clamp 在放置区 + 避开障碍/基地 footprint。touch/mouse 通用。
3. 「开打」按钮 → `_dl_state="fight"` 启动。上/下/终极三战场各放置一次（终极放全部幸存）。
4. **验**：截图放置区、拖动落位、开打进战斗。1 commit。

## Phase E — 蛋 idle 修正 + 破碎死亡动画（PixelLab）
1. **先修 idle bug**：`_egg_sprite_dict` @1306 改成 `hframes=3, vframes=1, frames=3, fps~5, loop=true` → 单蛋 79×80 会轻晃（不再并排显 3 蛋）。同步蛋 billboard 宽高按 79×80 单帧比例（原按 237×80 会太扁）。
2. PixelLab 生成蛋 **裂→碎逐帧**(4-6帧，风格对齐 egg.png 单蛋 79×80，裂纹→碎壳飞溅) → `assets/sprites/map/egg_shatter.png`（横排 sheet）。
3. 接蛋死亡钩子（`_dl_flow_check` 蛋 death 处 @918-925）：播碎裂帧序列 billboard（播一次不循环）+ 碎片粒子 + 震屏，而非直接消失。
4. **验**：截图/连拍核对 idle 单蛋会动 + 死亡碎裂播放一次。1 commit。
