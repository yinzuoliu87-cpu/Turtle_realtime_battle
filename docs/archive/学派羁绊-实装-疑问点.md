## 疑问点

### Q1 — 小将/龟蛋等级 = 选龟平均等级(未接通)
**用户定**:深海小将等级 = 选龟时队伍平均等级,**固定**(不随局内 buy经验 涨)。

**代码现状(待核实/修)**:
- `setup_dual_lane(player_ids, enemy_ids, start_level)` 会把 `dual_level = start_level` 且 `egg_hp = egg_hp(start_level)`,注释说 `start_level` 应是队伍平均等级。
- **但 `setup_dual_lane` 只在冒烟测试 `BattleScene.gd:172` 调用**(硬编码 start_level=1)。
- 真实流程(MainMenu→Matchmaking→TeamSelect→DualLaneMap→Battle)只调 `reset_dual_lane()` → `dual_level={left:1,right:1}`,**平均等级从未计算/传入**。
- 后果:真实对局里 `dual_level` 恒从 1 起 → 小将永远 Lv1、龟蛋 HP 恒 `egg_hp(1)=1050`,与队伍实际等级无关。

**需用户拍板 / 待实现**:
- (a) 进双路时计算玩家队 + 敌队平均 `_level` → 作为 `start_level`(选龟平均等级)。
- (b) 小将用这个**固定**平均级(`make_minion(avg_lv)`),不用会涨的 `dual_level`。
- (c) 龟蛋:用户报过"升级要涨血"→ 龟蛋 HP **随 dual_level 涨**(已修 _resync);但起始也应是 `egg_hp(avg_lv)` 而非 `egg_hp(1)`。即龟蛋起点=平均级,之后随局内升级涨;小将固定平均级。确认这个区分?

**✅ 已实装(本会话)**:
- `GameState.dual_avg_level`(固定)+ `team_avg_level(ids)`;`reset_dual_lane` 重置。
- `DualLaneMapScene`:进双路时算 `team_avg_level(picked)` → 设 `dual_level` 起始 + `dual_avg_level`(固定)+ `egg_hp(avg_lv)`(原硬编 egg_hp(1))。
- `_fill_lane_minions`:小将用 `dual_avg_level`(固定平均级),不再用会涨的 `dual_level`。
- 龟蛋:起始 `egg_hp(avg_lv)`,之后随 `dual_level`(buy经验)涨(`_reinforce_egg` + 登场后 `_resync_spawned_egg_hp`)。
