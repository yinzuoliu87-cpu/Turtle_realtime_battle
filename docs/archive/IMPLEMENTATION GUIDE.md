# 深海龟战 · 实装指引（给 Claude Code 看）

> 这份文件是写给在 VS Code / Claude Code 里帮忙实装的 Claude 的。
> 目的：让你（Claude）知道**怎么分步把本轮新设计接进现有游戏**，哪些是确定逻辑、哪些是占位待调。
> 配套主文档：《深海龟战-总设计文档.docx》（最终决策依据）。本指引只讲“怎么实装”，规则细节以总设计文档为准。

-----

## 0. 最重要的前提

1. **这是往一个已有的游戏里加功能，不是从零做。** 统领系统、伤害公式、回合框架（我方出手/敌方出手）、选将等已经实现。动手前，先读现有代码，搞清楚：
- 回合循环在哪、怎么驱动（“1 大回合 = 我方出手 + 敌方出手”）
- 统领（龟）的数据结构、属性字段、技能挂载方式
- 现有伤害公式（攻击/护甲/魔抗/穿透怎么算）——新机制（×5 增伤、永恒 buff +50%）要挂到这个公式上
1. **不要一次实装全部。** 按下面的模块顺序，一块一块来，每块跑通再下一块。
1. **凡标【占位】的数值，先填能跑的临时值，等玩家实测调。** 不要假装它们是最终值。

## 1. 模块拆分与建议实装顺序

按依赖关系排序，从地基到上层：

### 模块 A — 装备与稀有度数据层（先做，是一切的地基）

- 把 40 件核心装备 + 约 28 件新建装备，建成数据表（见配套《装备数据表.md》）。
- 每件字段：`id / 名称 / 系列 / 所属根 / 所属线 / 层级 / 稀有度 / 属性加成 / 效果描述 / 进化指向`
- 稀有度 7 档：初始/普通/精良/稀有/史诗/传说/特殊。
- **依赖**：现有装备系统（如果已有装备基类，扩展它；没有就新建）。
- **输出**：可被进化系统、商店系统查询的装备数据库。

### 模块 B — 进化树 + 进化石

- 进化树结构：6 系列，每系列 2 根，每根 2~3 线，每线 2~3 层。数据见《装备数据表.md》的“进化指向”字段。
- 进化石逻辑：
  - 对一个“根/中阶”装备使用 → 从该件的可进化方向**随机亮 2 个** → 玩家二选一（硬随机）。
  - **roll 在“用石那刻”才生成**，无预览。
  - 一颗石推进一步（根→中阶 或 中阶→终态）。
  - 进化 = 替换：新件占同格、旧件消失、**该件局内成长完全重置**（哪些字段算“局内成长”要结合现有代码定义：叠层数、临时等级、统计值等）。
- 后悔缓冲：进化后可花深海币换方向 → 重新 roll 仍 2 选 1 → 每件**一生限 1 次** → 同根同层 → 同样重置成长。
- **使用时机**：轮到我方出手、每只龟出手前的 **30 秒倒计时窗口**内可用（进化石、物品、换装备都在此窗口）。
- **依赖**：模块 A。
- **占位**：进化石售价、换方向价。

### 模块 C — 经济（深海币）

- 单一钱包，每局重置。来源：每大回合获得 + 利息 + 单位阵亡。
- 花费：买装备、刷新商店、买进化石、换方向。
- **依赖**：无强依赖，但商店/进化要用到。
- **占位**：每回合产出、利息公式、阵亡奖励、各项花费——**全部占位，需实测**。

### 模块 D — 商店（两种）

- **小商店**：每 **4 个大回合**出现一次；5 商品位（卖装备+进化石）；花币刷新、价格递增（2→4→6…）；买不起显示“钱不够”；不能卖；走了就关、不可重进。
- **大商店**：上路打完→下路开打之间出现；只卖物品（消耗品）；只能买不能用。
- 买的东西都进**备战席（背包栏）**，容量 10。
- **依赖**：模块 A、C。
- **占位**：商品权重表、刷新涨价曲线。

### 模块 E — 备战席（背包栏）

- = 装备席 = 背包栏，同一个东西。常驻侧边，容量 10。
- 买来的装备/石/物品先进这里。
- 分配/用石/用物品都在**出手前 30 秒窗口**做。
- 装备到龟身上：拖拽（真机）；备战席满 10 不能买、龟槽满 **5** 不能装。
- **注意**：手机 HTML5 原生拖拽不可靠，真机用触摸拖拽库或“点选→点目标”方案。
- **依赖**：模块 A、D。

### 模块 F — 战斗模式扩展（双路 + 残局）

- 流程：选将 → 分路暗选（**分路即分死**，龟整局只在分到的那路）→ 上路战斗 → 下路战斗（**串行**）→（若双方蛋都活）终极战场。
- 小将补位：每路补到双方各 3 单位（前3+后3）。1 级小将 250血/30攻/7护甲/7魔抗，每级 ×1.05。前排挥砍 1.4×ATK、后排射击 1.5×ATK。某路 0 统领→1 前排小将升精英（有概率带小将击杀 1 敌方统领）。
- 龟蛋 HP = 1000 + 50 × 该方 3 龟平均等级。
- 攻蛋累计：上/下/终极对同一颗蛋的伤害持续累加，归零才碎。
- 每路结束：存活单位回复已损失生命的 30%，退场待命等终极。
- **依赖**：现有战斗框架（重点对接处）。

### 模块 G — 终极战场机制 + 永恒 buff

- 终极战场三情况：①两路各赢一路→双方幸存者汇合决战 ②一方两路通吃且已推穿蛋→当场胜 ③通吃但没推穿→幸存者独自凿蛋。
- 败方蛋登场后（仅此阶段）：蛋受 **×5 增伤** + 每回合**自损 25% maxHP**。
- 结算节奏：回合交替（我方/敌方出手），蛋拥有方回合自损、敌方回合挨打，无平局。
- **永恒 buff（反僵持）**：每场战斗（上/下/终极）各自独立计回合，**第 30 大回合后**，每只存活**统领**每回合叠 1 层，每层造成+受到伤害各 +50%（线性叠加）。**只加统领，小将和龟蛋不吃。** 层数每场独立，进新战场清零重算。
- **依赖**：模块 F、现有伤害公式。

## 2. 哪些是确定逻辑 vs 占位待调

|类别    |确定逻辑（照实装）                   |占位（先填临时值，实测调）   |
|------|----------------------------|----------------|
|装备    |稀有度分档、进化树结构、进化=替换重置         |28 件新建装备的具体属性/效果|
|进化石   |roll 2选1、用石那刻roll、一颗一步、换方向规则|售价、换方向价         |
|经济    |单一钱包、来源种类、每局重置              |各项产出/花费的具体数字    |
|商店    |两种商店、刷新递增、备战席容量10、龟槽5       |商品权重表、涨价曲线      |
|战斗    |分路即分死、串行、累计伤害、回复30%、补小将规则   |小将精英击杀概率        |
|终极    |三情况判定、×5、自损25%maxHP、无平局     |（倍率/比例已定，可直接用）  |
|永恒buff|每场独立数30大回合、+50%线性、仅统领       |30 这个门槛（防呆保险，可调）|

-----

## 3. 实装时的注意点

- **新机制挂到现有伤害公式上**：×5 增伤、永恒 buff +50% 增/受伤——要确认它们和现有护甲/魔抗减伤的**运算顺序**（建议：先算基础伤害→应用增伤倍率→再过减伤，但以现有公式实际为准，别破坏已实现的平衡）。
- **“局内成长重置”要逐字段确认**：进化替换时重置哪些（叠层、临时等级、统计），别误伤永久属性。
- **大回合是全局节奏基准**：商店触发（每4大回合）、永恒buff（每场第30大回合）都按大回合数。确认现有回合计数器能区分“全局大回合”和“单场回合”。
- **每个模块跑通再下一块**，不要堆一起。建议先 A→B（装备+进化石能在战斗里用起来），这是本轮设计的核心爽点，最该先验证手感。
