## 三个事实锚（最重要）
1. **XP ↔ 币干净可拆**：`grant_dual_round`（`GameState.gd:172-181`）里 XP（176-177）与 币/利息（178-181）是两段独立循环体；`add_xp`/`_reinforce_egg` 不读币；唯一耦合是 `buy_xp` 的扣币行。删 178-181、保 176-177 即解耦。
2. **删 `dual_coins` 的安全做法 = 不删字段、只断产出口**。余额恒 0 → 所有读点（AI 购物 `:417`、快照 `:580/598`、HUD pill、买/刷按钮）安全置灰、零崩溃。删字段会连带改快照 + 10+ 读点 = 高风险，**不做**。
3. **商店算法值钱、留给局外**：`try_merge_all`/`try_merge_bench`/`buy_shop_item`/`roll_shop_offer`/`sell_*`/`equip_to_turtle` 是纯逻辑无 UI 耦合，阶段 2 局外商店直接复用 —— **阶段 1 只删"战斗内呈现 + 调用"，不删算法**。
